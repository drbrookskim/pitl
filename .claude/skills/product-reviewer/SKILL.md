# Product Reviewer Skill

1-pager 요약서, PRD 명세서, 와이어프레임 3안 시안의 유기적 정합성을 감사하여 비즈니스 논리의 누락이나 디자인과 개발 요구사항 간의 어긋남을 원천 방지하는 **정합성 검수(QA)** 스킬입니다.

## 핵심 검수 절차

### 1. 비즈니스-명세-UX 삼각 검증 (Cross-Matching)
- **1-pager ↔ PRD**: 1-pager에 명시된 비즈니스 문제 정의가 PRD의 KPI로 올바르게 수치화되었으며, 문제의 Root Cause를 해결하기 위한 기능(Phase 1, REQ ID)들이 PRD 상세 요구사항 테이블에 완전히 존재합니까?
- **PRD ↔ 와이어프레임**: PRD에 명시된 핵심 기능 ID(REQ-01 등)와 사용자 예외 정책(로딩, 실패, 빈 화면 처리)이 와이어프레임 3안의 레이아웃 구성 요소로 완전히 실체화되어 있습니까?

### 2. 보안 및 커맨드 안전 점검
- 쉘 메타 문자를 활용한 악의적인 파일 저장 공격을 사전 무력화하는 파일명 필터링 규칙이 PRD 및 관련 스크립트에 명문화되어 있는지 검사합니다.

### 3. 검수 보고서 발행 (`_workspace/04_reviewer_audit.md`)
아래 구조를 완벽하게 유지하여 보고서를 작성합니다:

```markdown
# [정합성 검수 보고서] {프로젝트명}

- **최종 판정**: [PASS] 또는 [REVISION NEEDED]
- **감사 일시**: {YYYY-MM-DD}T{HH:MM}+09:00

---

## 1. 정량적 일치성 매칭률

| 감사 항목 | 검증 사항 | 점검 결과 | 판정 이유 |
| :--- | :--- | :--- | :--- |
| **비즈니스 정합성** | 1-pager 문제 정의가 PRD 요구사항에 완전히 반영됨 | [Y / N] | - |
| **기능-UX 연결성** | PRD 요구사항 REQ ID가 와이어프레임 CTA에 매핑됨 | [Y / N] | - |
| **예외 정의 충족** | 로딩/실패/빈 상태 UI 정책이 명시 및 시각화됨 | [Y / N] | - |
| **보안 안전성** | 파일 저장 필터링 및 덮어쓰기 로직 명문화됨 | [Y / N] | - |

---

## 2. 정성적 피드백 및 Revision 대안
- **지적 사항**: {REVISION NEEDED인 경우 구체적 모순점 기재}
- **수정 권장안**:
  ```diff
  - {기존 누락되거나 어색한 요구사항 및 정책}
  + {보강된 정교한 정책 요구사항 또는 레이아웃 요소}
  ```

---

## 3. 최종 결재
