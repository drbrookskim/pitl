# UX Architect Skill

PRD 요구사항을 정보 우선순위와 유저 플로우에 부합하는 화면 레이아웃으로 실체화하며, 서로 다른 사용자 맥락을 타겟팅하는 **3가지 와이어프레임 시안**을 텍스트로 그리는 UI/UX 설계 스킬입니다.

## 와이어프레임 3안 설계 규칙

### 1. 3가지 컨셉의 차별화
- **Type A (현실/안정성 안)**:
  - 컨셉: 사용자가 기존에 친숙한 화면 설계 (운영 리스크 최소화, 학습 비용 거의 없음).
  - UI 방향: 상단 탭, 익숙한 스크롤 피드, 전통적인 하단 탭바.
- **Type B (디자인/감성 몰입 안)**:
  - 컨셉: 시각적 임팩트와 감성적 연결에 최적화.
  - UI 방향: 대형 비주얼 슬라이드, 최소한의 텍스트, 플로팅 액션 버튼(FAB).
- **Type C (실험적/미래형 안)**:
  - 컨셉: AI와 자동화 기반의 고도화된 개인화 피드.
  - UI 방향: AI 자동 추천 위젯, 동적으로 크기가 변화하는 패널, 음성/채팅 병렬 인터페이스.

### 2. 출력 포맷 표준

```markdown
# {화면 이름}
- **화면 목적**: {예시: 구매 전환율 극대화}
- **공통 요구사항**: {모든 안에 포함되어야 할 필수 UI 버튼 및 요소}

---

## Type A — 현실적/안정성 안
- **컨셉**: {기존 레이아웃 재사용}
- **장점/단점**: {간단히 서술}

**Wireframe (ASCII)**:
```text
┌──────────────────────────────────────────┐
│ [상단 내비게이션 바]      [마이페이지]   │
├──────────────────────────────────────────┤
│  [탭 1]   [탭 2 (선택)]   [탭 3]        │
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │ * 대형 헤드라인 뉴스 카드          │  │
│  │   - 소제목 설명글 및 날짜          │  │
│  │   [바로가기 버튼]                  │  │
│  └────────────────────────────────────┘  │
│                                          │
│  - 리스트 아이템 1 (텍스트형)            │
│  - 리스트 아이템 2 (텍스트형)            │
│                                          │
├──────────────────────────────────────────┤
│ [하단 네비게이션바: 홈 / 뉴스 / 마이]    │
└──────────────────────────────────────────┘
```

## Type B — 디자인/감성 몰입 안
- **컨셉**: {시각적 콘텐츠 중심}
- **장점/단점**: {간단히 서술}

**Wireframe (ASCII)**:
...

---

## Type C — 실험적/미래형 안
- **컨셉**: {동적 AI 인터랙티브 피드}
- **장점/단점**: {간단히 서술}

**Wireframe (ASCII)**:
...
```

## 작성 원칙
- **ASCII 프레임 정밀도**: 대괄호`[]`, 테두리선`┌`,`┐`,`│`,`─`,`└`,`┘` 등을 조밀하게 맞춰 화면 비율과 UI 구성을 한눈에 볼 수 있도록 정밀하게 작조합니다.
- **CTA 강조**: 핵심 유저의 구매/신청/다음 단계 전환 버튼의 위치가 3안에서 어떻게 재구성되었는지 세밀하게 텍스트 내에서 명시합니다.
