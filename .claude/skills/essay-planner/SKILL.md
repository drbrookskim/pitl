# Essay Planner Skill

Brunch 에세이의 핵심 주제를 정의하고, 기존 사유 연결망과 결합할 수 있도록 철학적·논리적 뼈대를 설계하는 스킬입니다.

## 핵심 기획 절차

### 1. 도메인 및 주제 탐색
- 사용자가 던진 대략적인 화두나 주제를 분석합니다.
- 기존 아티클 카테고리 인덱스 파일들(`../기획론.md`, `../AI와기술.md`, `../사고와언어.md` 등) 및 `../brunch_articles/` 아래의 아티클 제목을 검색하여 사용자가 던진 주제가 기존 어떤 생각 그래프와 긴밀하게 호흡할 수 있는지 확인합니다.

### 2. 7대 카테고리 결정 및 명제 수립
- 글이 속할 단 하나의 카테고리를 결정합니다:
  - **기획론**: 기획의 본질, 메타 인지, 상수 제어와 변수 관리, 핍진성 등.
  - **상품기획**: 버티컬의 방향성, 사용자 맞춤, 성공과 실패 등.
  - **AI와 기술**: 인공지능과 기술의 진화, 프롬프팅, 도구의 명령어화 등.
  - **인간과 심리**: 대니얼 카너먼, 행동 경제학, 내면의 소리, 직관성 등.
  - **사고와 언어**: 생각의 깊이, 기호와 상징, 언어의 진화 등.
  - **관계와 사회**: 인간 관계, 사회적 항상성, 지리적 결정론 등.
  - **경제와 가치**: 자산으로서의 아이디어, 측정되지 않는 가치 등.
- 에세이가 도달하고자 하는 단 한 문장의 **'핵심 명제'**를 수립합니다.

### 3. 지식 그래프 엣지(Edge) 매핑
- 새로운 글이 독자에게 제공할 지적 탐험의 깊이를 극대화하기 위해, 기존 아티클 중 가장 깊게 연결되는 **최소 2개 이상의 아티클**을 선정합니다.
- 예: "언어가 도구에서 명령어가 되는 순간" 에세이를 기획할 경우, "Prompting으로 기획하기" 아티클과 "사고와 언어"에 관한 다른 글들을 연결합니다.

### 4. 구조화된 아웃라인(Outline) 작성
- 기획론적 서술 구조인 **[개념의 재해석 → 현실의 시나리오/사례 → 철학적 성찰 및 궁극 요인 탐색]**의 3단 흐름으로 분해합니다.
- 각 장의 소제목과 함께, 해당 단락에서 전개될 핵심 사유 메커니즘을 상세히 기록합니다.

## 산출물 규격 (`_workspace/01_planner_structure.md`)

```markdown
# [기획 초안] {에세이 제목}

- **카테고리**: {7대 카테고리 중 택1}
- **핵심 명제**: {에세이가 관통하는 단 하나의 명제}
- **연결 대상 아티클 (지식 그래프 엣지)**:
  1. `brunch_articles/{filename_1}.md` ({이유})
  2. `brunch_articles/{filename_2}.md` ({이유})

---

## 1. 도입: {소제목}
- **개념**: {설명할 본질적 개념}
- **핵심 메시지**: {전달할 가치}

## 2. 전개: {소제목}
- **시나리오**: {일상적 또는 비즈니스적 구체적 사례}
- **핵심 메시지**: {인사이트}

## 3. 결론: {소제목}
- **궁극 요인**: {단순 하우투를 넘어서는 궁극적 성찰}
- **핵심 메시지**: {독자에게 남기는 사유의 여운}
```
