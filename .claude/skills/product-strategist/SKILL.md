# Product Strategist Skill

사용자의 거칠거나 불완전한 상품 아이디어를 분석하여 비즈니스 목적과 고객 문제를 선명하게 분해한 **1-pager 요약서**를 작성하는 기킬입니다.

## 1-pager 요약서 작성 프로세스

### 1. 아이디어 정렬 및 문제 탐색
- 사용자가 해결하고자 하는 문제를 파악합니다.
- 다음 항목을 명시적으로 추출합니다:
  - **고객 문제**: 실사용자가 느끼는 명백한 불편함.
  - **비즈니스 문제**: 회사나 팀이 비즈니스 생존/확장을 위해 반드시 해결해야 할 점.
  - **데이터 현황**: 가설의 타당성을 입증할 정량 지표 (가상 수치라도 합리적인 한계 내에서 설정 가능).

### 2. 표준 1-pager 템플릿 작성
작성 시 아래 구조와 표 형식을 엄격하게 준수합니다:

```markdown
# [{YY.MM.DD}] {프로젝트 제목} ({팀/조직명})

## 1. 프로젝트 한 줄 요약
- {짧고 강렬하게 핵심 솔루션을 설명}

## 2. 대상
| 대상 | 상황/특징 |
| :--- | :--- |
| {타겟 사용자 그룹} | {그들의 현재 맥락 및 페인 포인트} |

## 3. 문제 정의
### 고객 문제
- {고객 관점에서의 불편사항 목록}

### 비즈니스 문제
- {비즈니스/수익/운영 관점에서의 손실 및 병목 목록}

### 데이터 기반 현황
| 주요 메트릭 | 현재 수치 | 인사이트 |
| :--- | :--- | :--- |
| {지표명} | {X%} | {수치가 나타내는 현황 설명} |

### Root Cause
- **Case 1**: {근본 원인 1}
- **Case 2**: {근본 원인 2}

## 4. 목표
| 항목 | 목표 수치 |
| :--- | :--- |
| {측정 가능한 정량 목표} | {달성하려는 구체적 % 또는 수치} |

## 5. 현황 및 분석
- {VOC 인용, 정성 인터뷰 기록, 혹은 시장 동향 내용 요약}

## 6. 해결방안 (Phase 단위 기획)
- **Phase 1**: {초기 최소 기능 구현 및 가치 검증}
- **Phase 2**: {성능 고도화 및 확장 단계}

## 7. 리스크
| 영향도 | 리스크 | 대응 방안 |
| :--- | :--- | :--- |
| [상/중/하] | {발생 가능한 문제 상황} | {위험 우회 또는 해결 시나리오} |

## 8. 중요도 및 긴급도
- **중요도**: [상/중/하] (이유: {구체적 서술})
- **긴급도**: [상/중/하] (이유: {구체적 서술})
```

## 작성 원칙
- **Why First**: 기능 정의가 아닌, 이 기획이 왜 필요한가에 집중하여 작성합니다.
- **수치 명세**: 목표와 문제는 반드시 측정 가능한 형태로 서술되어야 합니다.
- **보안 검증**: 파일 저장 요구 시 쉘 메타 문자 제거 및 덮어쓰기 여부 질문을 수행해야 합니다.
