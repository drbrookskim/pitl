# 상품기획 통합 프레임워크
## 오마에 겐이치 × 3C × MECE

이 스킬은 세 가지 프레임워크를 통합해 상품기획 전 단계를 체계적으로 지원한다.

---

## 1. 오마에 겐이치의 문제해결 사고법

### 핵심 철학
> "전략은 분석에서 나오지 않는다. 분석을 통해 제약을 이해하고, 그 제약 안에서 창의적으로 생각하는 것이 전략이다."

오마에는 맥킨지 수석 컨설턴트로서 분석적 사고를 넘어 **창의적 통찰**로 문제의 본질을 꿰뚫는 방법론을 제시했다. 핵심 사고 흐름: **Why → So What → How**

### 4대 사고 원칙

#### 1) 이슈 드리븐 사고 (Issue-Driven Thinking)
- 정답보다 **옳은 질문**이 먼저다
- 문제를 재정의(Reframe)하여 진짜 이슈를 찾는다
- "매출이 왜 떨어졌는가"가 아니라 "우리가 해결해야 할 이슈는 무엇인가"로 시작
- 도구: 이슈 트리, 5 Whys, 문제 재정의 워크샵

#### 2) 가설 → 검증 → 통찰
- 데이터를 수집한 뒤 분석하는 것이 아니라, **먼저 가설을 세우고** 그를 검증하는 데이터를 찾는다
- 빠른 가설-검증 사이클이 통찰 속도를 높인다
- Fact → Insight → Implication의 흐름
- 데이터를 보고하는 것과 통찰을 제공하는 것은 다르다

#### 3) MECE 분해와 구조화
- 복잡한 문제는 **독립적이고 완전한 부분**으로 분해한다
- 중복 없이(Mutually Exclusive), 누락 없이(Collectively Exhaustive)
- 로직 트리(Logic Tree), 이슈 트리(Issue Tree)로 시각화

#### 4) KSF 집중 (Key Success Factor)
- 모든 요인이 동등하지 않다
- KSF를 찾아내 거기에 자원을 집중한다
- 오마에는 이를 "전략적 자유도의 영역"이라 불렀다
- 80/20 원칙: 20%의 분석이 80%의 인사이트를 만든다

### 7단계 문제해결 프로세스

| 단계 | 명칭 | 핵심 행동 |
|------|------|-----------|
| 1 | 문제 정의 | 현상이 아닌 근본 문제(root cause) 탐색. 5 Whys 적용 |
| 2 | 가설 수립 | 분석 전 명시적 가설 수립. 검증 방향 설정 |
| 3 | MECE 분해 | 로직 트리로 문제를 중복·누락 없이 분해 |
| 4 | 핵심 분석 | KSF 직결 2~3개 분석에만 집중. 80/20 적용 |
| 5 | 통찰 도출 | So What? — 분석 결과에서 행동 함의 추출 |
| 6 | 대안 생성 | 인사이트 기반 실행 가능한 대안 생성 (창의적 도약) |
| 7 | 의사결정 | 70% 정보로 결정하고 실행 중 수정. 완전한 정보를 기다리지 않는다 |

## 2. 3C 분석 프레임워크

오마에가 직접 정립한 전략 삼각형. 모든 전략적 의사결정은 세 주체의 관계 속에서 이루어진다.

### 전략 공식
> **지속 가능한 차별화 공간 = 고객이 원하고(C1) + 경쟁자가 못 하며(C2 Gap) + 우리가 잘 할 수 있는(C3 Strength) 교집합**

### Customer (고객) — 수요 측면

**분석 항목:**
- **고객 세분화**: 인구통계 / 심리통계 / 행동 / 니즈 기반 세그먼트
- **니즈 계층화**: 기능적 니즈 / 감성적 니즈 / 잠재적 니즈
- **구매 의사결정 여정**: 인지 → 탐색 → 비교 → 구매 → 재구매
- **지불 의향(WTP)**: 최대 지불 의향 가격, 가격 민감도 분석
- **Jobs-to-be-Done**: 고객이 제품을 "고용"하는 진짜 이유

**핵심 질문:** 어떤 고객이, 어떤 상황에서, 무엇을 원하는가?

**함정 경고:** 고객을 충분히 이해하지 못하면 기술 중심의 상품이 탄생한다. 표면적 니즈 뒤의 진짜 동기를 발굴해야 한다.

### Competitor (경쟁자) — 경쟁 측면

**분석 항목:**
- **경쟁 지형도**: 직접경쟁 / 간접경쟁 / 잠재 진입자 / 기술 대체재
- **경쟁사 강·약점**: 제품, 가격, 유통, 마케팅, 기술, 자본력
- **포지셔닝 맵**: 주요 속성 2축으로 경쟁 공간 시각화
- **경쟁 전략 유형**: 원가우위 / 차별화 / 집중화

**핵심 질문:** 경쟁자 대비 우리의 차별화 포인트는 무엇인가?

**함정 경고:** 직접 경쟁자만 보는 것이 함정. 고객이 현재 문제를 해결하는 모든 방법(엑셀, DIY 포함)이 경쟁이다.

### Company (자사) — 역량 측면

**분석 항목:**
- **핵심 역량**: 기술, 브랜드, 채널, 데이터, 생산 능력
- **가치 사슬 분석**: R&D → 제조 → 마케팅 → 판매 → AS
- **자원 제약**: 예산, 인력, 시간, 기술 투자 현황
- **전략적 방향성**: 비전, 포트폴리오 전략, 투자 우선순위
- **확보 가능 역량**: 현재뿐 아니라 12~18개월 내 확보 가능한 역량

**핵심 질문:** 우리가 실현할 수 있는 역량과 자원은 무엇인가?

**함정 경고:** 역량을 절대적으로 평가하지 말고 "경쟁자 대비" 상대적으로 평가해야 한다.

### 3C × 상품기획 연결 매트릭스

| 구분 | 핵심 질문 | 분석 도구 | 상품기획 산출물 |
|------|-----------|-----------|----------------|
| Customer | 어떤 고객이 무엇을 원하는가 | JTBD, 고객 여정 맵, 인터뷰 | 타겟 페르소나, 사용 시나리오 |
| Competitor | 차별화 포인트는 무엇인가 | 포지셔닝 맵, 기능 비교표 | 차별화 포인트, 포지셔닝 방향 |
| Company | 실현 가능한 역량은 무엇인가 | 역량 평가, 기술 로드맵 | MVP 범위, 개발 우선순위 |
| 교집합 | 지속 가능한 기회는 어디인가 | 기회 스코어링, VPC | 제품 컨셉, 가치 제안 확정 |

## 3. MECE 구조화 원칙

**Mutually Exclusive**: 분석 항목 간 중복이 없다
**Collectively Exhaustive**: 전체를 빠짐없이 포괄한다

### 상품기획 5대 MECE 분해 프레임워크

#### A. 고객 세분화 MECE
```
전체 고객
├── 인구통계적: 연령 / 성별 / 소득 / 지역 (겹치지 않고 전체 커버)
├── 니즈 기반: 가격 민감 / 품질 중시 / 편의 추구 / 혁신 선도
└── 라이프스테이지: 학생 / 사회초년생 / 중장년 / 시니어
```
검증: 각 세그먼트가 겹치지 않는가? + 모든 잠재 고객이 어딘가에 속하는가?

#### B. 고객 가치 MECE
```
고객 가치
├── 기능적 가치: 실제 수행 기능, 성능, 편의성
├── 경제적 가치: 비용 절감, 효율 향상, ROI
├── 감성적 가치: 브랜드, 신뢰, 자아 표현, 소속감
└── 사회적 가치: 환경, 윤리, 공동체 기여
```

#### C. 시장 기회 MECE (앤소프 매트릭스)
```
시장 기회
├── 기존 제품 × 기존 시장 = 시장 침투 전략
├── 신제품 × 기존 시장 = 제품 개발 전략
├── 기존 제품 × 신시장 = 시장 개발 전략
└── 신제품 × 신시장 = 다각화 전략
```

#### D. 기능 우선순위 MECE (MoSCoW)
```
전체 기능
├── Must Have: 없으면 제품 자체가 성립하지 않는 핵심 기능
├── Should Have: 있어야 경쟁력 있는 주요 기능
├── Could Have: 있으면 좋지만 없어도 되는 부가 기능
└── Won't Have: 이번 버전에서 의도적으로 제외하는 기능
```

#### E. 리스크 MECE
```
전체 리스크
├── 시장 리스크: 수요 예측 오류, 경쟁 격화, 트렌드 변화
├── 기술 리스크: 개발 지연, 성능 미달, 보안 취약점
├── 운영 리스크: 공급망, 제조, QA, 고객 지원
├── 재무 리스크: 예산 초과, 투자 회수 지연, 환율
└── 규제 리스크: 법령 변화, 인증, 개인정보 규제
```

## 4. 통합 프로세스 — 5개 Phase

| Phase | 명칭 | 오마에 적용 | 3C 렌즈 | MECE 적용 | 핵심 산출물 |
|-------|------|------------|---------|-----------|------------|
| 1 | 기회 탐색 | 이슈 정의, 가설 수립 | Customer 니즈, Competitor 공백 | 시장 기회 MECE | 기회 영역 정의서 |
| 2 | 시장 분석 | 구조 분해, 핵심 분석 | 3C 심층 분석 | 고객/경쟁 MECE | 시장 분석 리포트 |
| 3 | 컨셉 개발 | 통찰 도출, 창의적 대안 | 3C 교집합 도출 | 기능 MECE 분류 | 제품 컨셉 정의서 |
| 4 | 상세 기획 | KSF 집중, 80/20 | Company 역량 검증 | 기능/리스크 MECE | PRD, 로드맵 |
| 5 | 의사결정 | 70% 정보로 결정 | 3C 재검토 | KPI 구조화 | 사업계획서, 출시 플랜 |

**핵심 원칙:** 이 프로세스는 선형이 아니다. Phase 3에서 Phase 1의 기회 정의가 바뀔 수 있고, 이는 프레임워크가 제대로 작동하고 있다는 신호다.

## 5. 상품기획 실전 도구

### 가치 제안 캔버스 (Value Proposition Canvas)

**고객 프로파일 (3C Customer 연동)**
- Jobs: 고객이 해결해야 할 과제
- Pains: 장애물과 불편
- Gains: 기대하는 혜택

**가치 맵 (Company 역량 연동)**
- Products & Services: 제공할 제품
- Pain Relievers: 고통 해소 방법
- Gain Creators: 가치 창출 방법

Fit 검증: 모든 핵심 Job이 Products에 매핑되는가? 모든 주요 Pain이 Pain Reliever에 대응하는가?

### 기능 우선순위 스코어링

| 기준 | 가중치 | 측정 방법 | 연결 프레임워크 |
|------|--------|-----------|----------------|
| 고객 가치 (Impact) | 40% | 사용자 인터뷰, 행동 데이터 | Customer KSF |
| 전략적 정합성 | 25% | 비전/OKR 기여도 | Company 방향성 |
| 경쟁 차별화 | 20% | 경쟁사 갭 분석 | Competitor 렌즈 |
| 실현 가능성 | 15% | 개발 공수, 기술 리스크 | Company 역량 |

### 시장 기회 평가 매트릭스 (2×2)

```
              높은 시장 매력도    낮은 시장 매력도
강한 자사 역량   ⭐ 핵심 투자 영역   🔧 역량 레버리지
약한 자사 역량   🏗️ 역량 개발 필요  🚫 회피 또는 철수
```

### 수익 모델 MECE 분류

- **반복 수익**: 구독, SaaS, 멤버십, 라이선스
- **거래 수익**: 단건 판매, 수수료, 마진
- **파생 수익**: 광고, 데이터, 제휴, 프리미엄

## 6. 마스터 체크리스트 (27개 항목)

### 오마에 — 이슈 정의 (4개)
- [ ] 핵심 이슈를 현상이 아닌 근본 원인으로 정의했는가
- [ ] 이슈 트리로 문제를 MECE하게 분해했는가
- [ ] 분석 전에 명시적 가설을 수립했는가
- [ ] 5 Whys로 이슈의 뿌리까지 파고들었는가

### Customer 분석 (5개)
- [ ] 타겟 고객 세그먼트를 MECE하게 정의했는가
- [ ] Jobs-to-be-Done 인터뷰를 수행했는가
- [ ] 기능적·감성적·사회적 니즈를 모두 파악했는가
- [ ] 고객 여정 맵(인지→추천)을 완성했는가
- [ ] 지불 의향(WTP) 및 가격 민감도를 측정했는가

### Competitor 분석 (4개)
- [ ] 직접 / 간접 / 잠재 경쟁자를 모두 식별했는가
- [ ] 포지셔닝 맵으로 경쟁 공백을 시각화했는가
- [ ] 경쟁사 제품의 강·약점을 객관적으로 분석했는가
- [ ] 잠재 기술 대체재(disruptor)를 고려했는가

### Company 역량 검토 (4개)
- [ ] 핵심 역량과 차별화 요인을 명확히 식별했는가
- [ ] 자원 제약(예산/인력/시간)을 현실적으로 파악했는가
- [ ] 12~18개월 내 확보 가능한 역량을 포함했는가
- [ ] 전략적 방향성과 제품 방향이 일치하는가

### MECE 구조화 (4개)
- [ ] 모든 분석 항목이 중복 없이 구분되는가
- [ ] 모든 중요 요소가 누락 없이 포함되었는가
- [ ] 기능 우선순위가 MECE 기준으로 분류되었는가
- [ ] 리스크가 5개 영역으로 MECE하게 식별되었는가

### 상품 컨셉 및 기획 (5개)
- [ ] 3C 교집합에서 차별화 포인트를 도출했는가
- [ ] 가치 제안 캔버스(Job-Pain-Gain 매핑)를 완성했는가
- [ ] MVP 범위가 KSF에 집중되어 있는가
- [ ] 수익 모델과 사업 모델이 정의되었는가
- [ ] KPI와 성공 지표가 MECE하게 설정되었는가

## 응답 가이드라인

이 스킬이 활성화되면 다음 방식으로 응답한다:

1. **질문 유형 파악**: 개념 설명 요청인가, 특정 Phase 지원인가, 전체 기획 지원인가
2. **프레임워크 선택**: 질문에 가장 적합한 프레임워크(오마에/3C/MECE/통합)를 우선 적용
3. **구체성 수준**: 사용자의 맥락(업종, 제품 유형, 기획 단계)에 맞춰 구체적 예시 제공
4. **산출물 제시**: 단순 설명이 아닌 실제 사용 가능한 형태(표, 체크리스트, 구조도)로 제공
5. **다음 단계 안내**: 현재 Phase에서 다음 Phase로 이어지는 행동을 명시

복잡한 기획 지원 요청 시에는 인터랙티브 HTML로 시각화한다.
