# [24.10.24] 폴더블 폼팩터 특화 기능 경험(Form-Factor UX) 극대화 프로젝트: 'FlexFlow' (Product Team)

## 1. 프로젝트 한 줄 요약
단순히 화면이 이어지는 '연속성(Continuity)' 경험을 넘어, 폴더블 기기의 물리적 접힘 상태(Form-Factor)를 감지하여 각 상태에 최적화된 독창적 기능 가치를 제공하는 UX 패러다임 전환 프로젝트.

## 2. 대상
| 대상 | 상황/특징 |
| :--- | :--- |
| **폴더블 기기 헤비 유저** | 멀티태스킹과 생산성을 중시하며, 일상 및 업무에서 Flex 모드(반쯤 접은 상태)를 자주 활용하고자 하는 사용자 |
| **일반 바형 스마트폰 이탈자** | 폴더블 기기를 구매했으나, 넓은 화면(대화면) 외에 폴더블만의 고유한 기능적 차별점을 체감하지 못하는 사용자 |
| **모바일 크리에이터 및 비즈니스 프로페셔널** | 별도 거치대 없이 기기를 접어 촬영하거나, 미팅 중 텐트 모드 등을 활용해 효율적인 정보 전달이 필요한 사용자 |

## 3. 문제 정의

### 고객 문제 (Customer Problem)
"비싼 가격을 지불하고 폴더블 폰을 구매했으나, 펼쳐서 크게 보는 것 외에는 일반 바(Bar)형 스마트폰과 기능적 차별점을 느끼지 못함." 화면 연속성은 편리하지만, 이는 '이동'의 개념일 뿐 폴더블 폼팩터만이 줄 수 있는 독점적 기능 경험(Utility Value)이 부족함.

### 비즈니스 문제 (Business Problem)
폴더블 시장의 성장 둔화 및 교체 주기 장기화 직면. 하드웨어 폼팩터의 혁신 대비 소프트웨어 경험의 진화 속도가 느려 프리미엄 고객의 재구매율(Retention) 저하 우려.

### 데이터 기반 현황 테이블
| 지표명 | 현재 수치 (As-Is) | 목표 수치 (To-Be) | 비고 |
| :--- | :--- | :--- | :--- |
| **Flex 모드(반쯤 접은 상태) 일평균 사용 시간** | 4.2분 | **15.0분** | 현재는 단순 유튜브 시청 및 카메라 타이머 촬영에 국한됨 |
| **커버-메인 화면 앱 연속성 만족도** | 78.0% | **85.0%** | 단순 화면 확대에 대한 만족은 높으나 기능적 변화 없음 |
| **폴더블 특화 기능(멀티윈도우 외) 주 1회 이상 사용률** | 12.5% | **35.0%** | 폼팩터의 물리적 변화를 활용한 기능 사용률이 매우 저조함 |

### Root Cause (근본 원인)
기존 UX 설계 사상이 '바형 스마트폰의 연장선(화면 확장 및 연속성)'에 머물러 있음. 기기의 물리적 상태 변화(접힘 각도, 거치 방향, 손에 쥐었는지 여부)를 능동적인 '기능 실행 트리거(Trigger)'로 활용하는 시스템적 UX 프레임워크가 부재함.

## 4. 목표
| 항목 | 목표 수치 |
| :--- | :--- |

| **폼팩터 기반 신규 기능(스마트 거치 제어 등) 활성화율** | 전체 폴더블 활성 유저의 **30.0% 이상** |
| **제품 추천 지수 (NPS, Net Promoter Score)** | 기존 대비 **+15pt** 상승 |

## 5. 현황 및 분석
* **고객 VOC 인용:**
  * *"화면이 안팎으로 부드럽게 이어지는 건 신기하지만, 그게 끝입니다. 접었을 때만 쓸 수 있는 유용한 도구들이 더 많아졌으면 좋겠어요." (30대 직장인 A씨)*
  * *"바닥에 반쯤 접어놓고 사진 찍을 때 빼고는 굳이 접어서 쓸 일이 없습니다. 접는 각도에 따라 앱 UI뿐만 아니라 기능 자체가 똑똑하게 변하면 좋겠습니다." (20대 대학생 B씨)*
* **시장 분석:** 경쟁사들 역시 '화면이 커지는 폴더블'에 집중하고 있어, '접히는 물리적 특성을 활용한 기능적 UX'를 선점할 경우 독점적인 브랜드 락인(Lock-in) 효과를 창출할 수 있음.

## 6. 해결방안

### Phase 1: 폼팩터 상태 감지 및 컨텍스트 매칭 엔진 구축 (기반 마련)
* **물리 상태 정밀 감지:** 힌지(Hinge) 각도 센서와 자이로 센서 데이터를 결합하여 기기의 상태를 5단계(닫힘, 미세 접힘, 직각 거치, 완만 거치, 완전히 펼침)로 세분화하여 감지하는 API 개발.
* **컨텍스트 인지:** 가속도 센서 및 조도 센서를 활용해 기기가 '바닥에 거치된 상태'인지 '손에 들려 있는 상태'인지 판별.

### Phase 2: 상태 기반 '액티브 펑션(Active Function)' UX 도입 (기능 구현)
* **Flex-Camera (90도 거치):** 기기를 접어 바닥에 놓으면, 상단은 뷰파인더로 전환됨과 동시에 하단은 단순 컨트롤러가 아닌 '타임랩스 편집기' 또는 '프로 모드 다이얼'로 기능적 역할 분담.
* **Flex-Office (텐트 모드):** 기기를 A자 모양(텐트 모드)으로 세우면, 전면(커버)에는 프레젠테이션 타이머와 슬라이드가 표시되고, 후면(메인)에는 발표자 스크립트가 자동으로 매칭되어 표시되는 비즈니스 특화 모드 실행.

### Phase 3: 서드파티 앱 생태계 확장 (확산)
* **FlexFlow SDK 배포:** 글로벌 주요 앱 파트너사(YouTube, MS Office, Zoom 등)에 폼팩터 상태 감지 SDK를 개방하여, 앱 자체적으로 접힘 상태에 따른 특화 기능을 구현하도록 유도.

## 7. 리스크
| 영향도 | 리스크 | 대응 방안 |
| :--- | :--- | :--- |
| **높음** | 센서 오작동으로 인한 의도치 않은 UI/기능 전환 (사용자 피로도 증가) | 각도 변화 후 1.5초간 상태가 유지될 때만 기능을 전환하는 '디바운스(Debounce) 알고리즘' 적용 및 퀵패널 내 '폼팩터 기능 잠금' 버튼 제공 |
| **중간** | 서드파티 앱의 폼팩터 UX 미지원으로 인한 일관성 저해 | 구글 안드로이드 가이드라인과의 정렬 및 주요 10대 앱 대상 전용 커스텀 매핑 프로필 자체 제공 |
| **낮음** | 센서 상시 대기로 인한 배터리 소모량 증가 | 저전력 센서 허브(Sensor Hub)를 활용하여 메인 AP의 개입을 최소화하고 백그라운드 연산 최적화 |

## 8. 중요도 및 긴급도
* **중요도: 상**
  * *이유:* 폴더블 카테고리의 존속 여부는 '왜 일반 폰 대신 폴더블을 사야 하는가'에 대한 답에 달려 있음. 하드웨어 스펙 경쟁을 넘어 독점적 UX 생태계를 구축하는 핵심 열쇠임.
* **긴급도: 상**
  * *이유:* 경쟁사들의 폴더블 시장 진입 가속화 및 유사 폼팩터(트라이폴드 등) 등장으로 인해, 기존 폴더블 라인업의 소프트웨어 차별화 골든타임이 향후 1년 이내로 제한됨. 빠른 선점이 필요함.
