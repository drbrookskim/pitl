# [24.10.24] AI 기반 저평가 우량주 발굴 플랫폼 'ValueFinder' PRD

## 0. 문서 개요
본 문서는 개인 투자자의 복잡한 재무 분석 과정을 3초 내로 단축하고, '밸류 트랩'을 방지하는 AI 기반 스크리닝 플랫폼 'ValueFinder'의 상세 요구사항을 정의합니다.

## 1. 목표 (Goal)
* **핵심 목표:** 데이터 기반의 직관적인 투자 의사결정 환경 제공을 통한 개인 투자자 시장 안착.
* **성공 기준:**
    * 종목 분석 시간: 4.5시간 → 3분 이내 단축
    * 가입 후 7일 이내 첫 스크리닝 완료율: 85% 이상
    * 유료 구독(Premium) 전환율: 6개월 내 4.5% 달성
    * 포트폴리오 초과 수익률: 시장 대비 연 +12.5%p

## 2. KPI 및 측정지표
| 구분 | 지표명 | 목표치 |
| :--- | :--- | :--- |
| **핵심 KPI** | 유료 구독 전환율 | 4.5% |
| **보조 지표** | 스크리닝 완료율 | 85% |
| **모니터링** | 일일 활성 사용자(DAU) | - |
| **품질 지표** | 데이터 정합성(API 오류율) | 0.1% 미만 |

## 3. 사용자 및 이해관계자
* **대상 사용자:** 스마트 초보 투자자, 바쁜 직장인, 퀀트 입문자.
* **제외 대상:** 초단타 매매(Scalping) 위주의 트레이더.
* **이해관계자:** Product Team(기획/디자인), Engineering(백엔드/AI), Data Vendor(금융 데이터 제공사), 법무팀(면책 조항 검토).

## 4. 정책 정의
* **기본 정책:** 모든 데이터는 투자 참고용이며, 최종 매매 결정은 사용자 본인에게 있음(Disclaimer 필수 노출).
* **상태값 정의:**
    * `Value Score`: 0~100점 (수익성 40%, 안전성 30%, 성장성 30% 가중치)
    * `Status`: Active(정상), Suspended(거래정지), Delisting(상장폐지 위험)
* **예외 정책:**
    * **결제 실패:** 결제 수단 오류 시 3회 재시도 후 구독 상태를 'Pending'으로 전환하고 사용자 알림.
    * **API 타임아웃:** 데이터 벤더 응답 지연 시 '데이터 업데이트 중' 메시지 노출 및 캐싱된 최신 데이터(최대 24시간 전) 우선 표시.

## 5. 서비스 구조 (User Flow)
1. **온보딩:** 투자 성향 선택 → 2. **대시보드:** 원클릭 필터 선택 → 3. **스크리닝:** AI Value Score 기반 종목 리스트 확인 → 4. **상세 분석:** 기업 재무/리스크 요약 확인 → 5. **액션:** 관심 종목 등록 또는 포트폴리오 시뮬레이션.

## 6. 상세 요구사항
| ID | 기능명 | 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| REQ-01 | Value Score | 재무 데이터 기반 100점 만점 점수 산출 | P0 |
| REQ-02 | 원클릭 필터 | 템플릿 기반 조건 검색 (고배당, 저PBR 등) | P0 |
| REQ-03 | AI 리스크 필터 | 밸류 트랩(사양 산업 등) 자동 제외 | P1 |
| REQ-04 | 백테스팅 툴 | 과거 데이터 기반 수익률 시뮬레이션 | P2 |

* **운영 요구사항:** 관리자 페이지 내 종목별 가중치 조정 기능, 공지사항 및 면책 조항 관리.
* **데이터 요구사항:** 일일 종목 재무 데이터 배치 업데이트(장 마감 후 1시간 이내).

## 7. UX/UI 고려사항
* **로딩:** 스크리닝 시 'AI가 종목을 분석 중입니다'라는 문구와 함께 프로그레스 바 노출.
* **실패:** API 오류 시 '현재 데이터를 불러올 수 없습니다. 잠시 후 다시 시도해주세요' 안내 및 재시도 버튼.
* **빈 화면:** 검색 결과가 없을 경우 '조건을 완화하여 다시 검색해보세요' 가이드 및 추천 필터 제안.

## 8. 기술 고려사항
* **데이터 보안:** 금융 데이터 벤더 API 통신 시 TLS 1.3 암호화 적용.
* **성능:** 대량의 종목 스크리닝을 위해 인메모리 데이터베이스(Redis) 활용하여 응답 속도 1초 이내 유지.

## 9. 실험 및 검증
* **A/B 테스트:** 필터 템플릿 UI(카드형 vs 리스트형)에 따른 스크리닝 완료율 비교.
* **검증:** 백테스팅 엔진 결과와 실제 시장 데이터 간의 오차율 1% 이내 유지 확인.

## 10. 리스크 및 대응
* **법적 리스크:** 모든 화면 하단에 "본 서비스는 투자 자문이 아닌 정보 제공 도구입니다" 고정 배너 노출.
* **데이터 오류:** 복수 벤더사 데이터 교차 검증(Cross-check) 로직을 통해 이상치(Outlier) 발생 시 관리자 알림 발송.
