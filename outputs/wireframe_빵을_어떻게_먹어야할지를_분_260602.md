UX Architect로서 'Bread-it' 서비스의 핵심 가치인 **'빵의 최적 상태 유지 및 조리 경험 최적화'**를 반영한 3가지 타입의 와이어프레임을 설계하였습니다.

---

### Type A: 현실/안정형 (Utility-First)
**특징:** 정보의 위계와 가독성을 최우선으로 하여, 사용자가 빠르게 조리법을 확인하고 타이머를 실행하는 데 집중한 구조입니다.

```text
┌──────────────────────────────────────────┐
│ [Bread-it]             [검색] [마이]     │
├──────────────────────────────────────────┤
│ 빵 종류: [ 식빵 ▼ ]  가전: [ 에어프라이어 ]│
├──────────────────────────────────────────┤
│ ┌──────────────────────────────────────┐ │
│ │ [ 이미지: 빵 상태 가이드 ]           │ │
│ └──────────────────────────────────────┘ │
│ ┌──────────────────────────────────────┐ │
│ │ 조리 설정: 180도 / 5분               │ │
│ └──────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│ [      타이머 시작 (CTA)      ]          │
└──────────────────────────────────────────┘
```

---

### Type B: 디자인/감성몰입형 (Experience-First)
**특징:** 빵의 질감과 조리 과정을 시각적으로 강조합니다. Lottie 애니메이션과 카드형 UI를 통해 홈카페의 감성을 자극합니다.

```text
┌──────────────────────────────────────────┐
│ [Bread-it]                     [설정]    │
├──────────────────────────────────────────┤
│ ┌──────────────────────────────────────┐ │
│ │                                      │ │
│ │      [ 빵이 구워지는 애니메이션 ]    │ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│ [ 빵 이름: 크루아상 ]                    │
│ [ 보관 상태: 냉동 ]                      │
├──────────────────────────────────────────┤
│ [ 180℃ ] [ 5분 ] [ 에어프라이어 ]       │
├──────────────────────────────────────────┤
│ [      지금 바로 굽기 (CTA)      ]       │
└──────────────────────────────────────────┘
```

---

### Type C: 미래형 동적 위젯 (Data-Driven)
**특징:** 사용자의 조리 이력과 가전 데이터를 기반으로 한 '맞춤형 위젯' 중심입니다. 실시간 상태 변화를 대시보드 형태로 제공합니다.

```text
┌──────────────────────────────────────────┐
│ [Bread-it]                   [알림: 2]   │
├──────────────────────────────────────────┤
│ ┌──────────────────────────────────────┐ │
│ │ [ 위젯: 현재 조리 상태 ]             │ │
│ │ 03:45 남음 (진행률: 25%)             │ │
│ └──────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│ [ 추천 가이드 ]                          │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│ │ [식빵]   │ │ [베이글] │ │ [바게트] │ │
│ └──────────┘ └──────────┘ └──────────┘ │
├──────────────────────────────────────────┤
│ [   가이드 보정 및 타이머 실행 (CTA)   ] │
└──────────────────────────────────────────┘
```

---

### UX Architect의 설계 의도 요약

1.  **Type A (현실/안정형):** 1인 가구 및 바쁜 사용자를 위해 **'검색-확인-실행'**의 단계를 최소화했습니다. 정보의 신뢰성을 강조하는 표준적인 레이아웃입니다.
2.  **Type B (디자인/감성몰입형):** 빵지순례족을 타겟으로 하여, 조리 과정을 하나의 **'경험'**으로 전환합니다. 시각적 만족도를 높여 앱 체류 시간을 증대시킵니다.
3.  **Type C (미래형 동적 위젯):** 데이터 기반의 개인화에 집중했습니다. 사용자의 이전 조리 이력을 학습하여 **'가장 최적화된 설정값'**을 위젯 형태로 즉시 제안함으로써 폐기율 감소라는 핵심 KPI를 달성합니다.

**공통 CTA:** 모든 타입은 하단에 고정된 **'타이머 실행(조리 시작)'** 버튼을 배치하여, 사용자가 가이드를 확인한 후 즉각적인 행동(Action)을 취할 수 있도록 설계하였습니다.
