# [26.05.31] 유동적 상품기획 자동화 플랫폼 (PITL-Process) (Product-Planning Team)

- **화면명**: 상품기획 자동화 대시보드 및 결과물 뷰어 (Desktop Web 기준)
- **화면 목적**: 사용자가 제출한 아이디어에 따른 기획 문서 3종(1-pager, PRD, 와이어프레임)을 교차 정합성 평가 점수와 함께 한 화면에서 다각도로 탐색하고 내보낼 수 있도록 정보 구조와 인터랙션을 설계합니다.
- **공통 요구사항**:
  - 최상단 브랜드 로고 및 현재 기획 히스토리 파일명 노출.
  - 정합성 통과 평가 배지 (예: `[QA PASS] Score: 98%`) 표시.
  - 마크다운(.md) 복사 버튼 및 PDF 내보내기 버튼 상시 탑재.

---

## Type A — 현실적/안정성 안 (전형적 문서 탭 뷰어 레이아웃)
- **컨셉**: 사용자에게 가장 친숙한 전통적 위키(Wiki) 및 개발 문서 구조 기반의 레이아웃.
- **UX 방향**: 좌측 사이드바 히스토리 탐색, 우측 상단 메인 액션(복사/다운로드), 본문 상단 탭 스위치.

**Wireframe (ASCII)**:
```text

│ [Logo] PITL-Process  |  현재 기획: 1pager_상품기획자동화   [QA PASS 98%]   │
├──────────────────────┬─────────────────────────────────────────────────────┤
│ (사이드바 - 히스토리)│ [ 탭: 1-pager ]   [ 탭: PRD ]   [ 탭: 와이어프레임 ]│
│ 🔎 [기획서 검색...]  ├─────────────────────────────────────────────────────┤
│                      │                                                     │
│ ■ 26.05.31_상품기획  │  # 1-pager: 상품기획 자동화 플랫폼                  │
│   - 1pager           │                                                     │
│   - prd              │  ## 1. 프로젝트 한 줄 요약                          │

│                      │  │ 대상                 │ 상황/특징              │  │
│                      │  ├──────────────────────┼────────────────────────┤  │
│                      │  │ 주니어 기획자 / PM   │ 문서 작성 시간 부족... │  │

│                      ├─────────────────────────────────────────────────────┤
```

- **특징**: 좌우 고정 그리드로 대량의 문서 정보 브라우징에 특화.
- **장점**: 직관적이고 별도의 학습 비용이 들지 않음. 기존 스프레드시트나 노션에 익숙한 기획자 선호.
- **단점**: 모바일 환경에서 좌우 스플릿 구조가 찌그러져 탭이 축소되는 경향이 있음.

## Type B — 디자인/감성 몰입 안 (비주얼 카드 및 와이드 리더 레이아웃)
- **컨셉**: 세련된 카드 뉴스 및 리서치 플랫폼 스타일의 모던한 슬라이드 및 와이드 리드 온리 레이아웃.
- **UX 방향**: 불필요한 메타 정보를 최소화하고 본문 폭을 넓혀 가독성을 최우선시하며, 하단에 Sticky 플로팅 배포 버튼(FAB)을 두어 시각적 몰입감을 배가.

**Wireframe (ASCII)**:
```text
┌────────────────────────────────────────────────────────────────────────────┐
│ [Logo] PITL-Process                   [ 기획서 히스토리 목록 ▾ ] [QA 98% ▾]│
├────────────────────────────────────────────────────────────────────────────┤

│      ┌──────────────────────────────────────────────────────────────┐      │
│      │                  # 1-pager: 상품기획 자동화                  │      │
│      │                                                              │      │
│      │  [ 슬라이드 카드 1: 문제 정의 ] ── [ 슬라이드 카드 2: 목표 ]│      │
│      │  ┌───────────────────────────┐   ┌────────────────────────┐  │      │

│      │                                                              │      │
│      │  [ ▾ 아래로 스크롤하여 상세 PRD 명세 읽기 ]                  │      │

│                                                                            │
│                                            ┌────────────────────────┐      │
```

- **특징**: 핵심 슬라이드 카드를 좌우 제스처로 쓸어넘긴 후 스크롤을 내리면 상세 PRD가 펼쳐지는 모던 캔버스 구조.
- **장점**: 프레젠테이션 및 아이디어 피칭 회의 시 시각적 전달력과 집중도가 탁월함.
- **단점**: 정보를 쪼개서 보기 때문에 긴 스펙 문서를 개발팀이 일괄 대조하여 스크롤하며 개발하기에 다소 불편할 수 있음.

## Type C — 실험적/미래형 안 (AI 에이전트 협업 라이브 콘솔 레이아웃)
- **컨셉**: AI 에이전트 팀원들(`strategist`, `analyst`, `architect`, `reviewer`)이 실시간 대화 및 태스크를 처리하는 인터랙티브 개발자 콘솔 스타일의 미래형 UI.
- **UX 방향**: 좌측에 기획 산출물 실시간 프리뷰, 우측에 에이전트 협업 콘솔 및 `SendMessage` 디버그 터미널 배치.

**Wireframe (ASCII)**:
```text
┌────────────────────────────────────────────────────────────────────────────┐
│ [Logo] PITL-Process               (라이브 콘솔 실행 모드)    [QA PASS 98%] │
├──────────────────────────────────────┬─────────────────────────────────────┤
│ (좌측 - 실시간 프리뷰 영역)          │ (우측 - 에이전트 팀 협업 콘솔)      │
│ ┌──────────────────────────────────┐ │ ┌─────────────────────────────────┐ │
│ │  # 1-pager                       │ │ │ [T1] 1-pager 기획 (strategist)  │ │
│ │  ## 1. 프로젝트 요약             │ │ │ [T2] PRD 명세     (prd-analyst) │ │
│ │  - 사용자가 입력한 한 줄의...    │ │ │ [T3] 와이어프레임 (ux-architect) │ │
│ │                                  │ │ │ [T4] 교차 검수   (reviewer)    │ │
│ │  # PRD                           │ │ ├─────────────────────────────────┤ │
│ │  ## REQ-01: 모드 지능 판별       │ │ │ (SendMessage 라이브 로그)       │ │
│ │  - 기능: 사용자가 제출한...      │ │ │ strategist: 1-pager 완공 전달.  │ │
│ │                                  │ │ │ reviewer: Dead-Link 감지, 보완.│ │
│ └──────────────────────────────────┘ │ └─────────────────────────────────┘ │
│                                      ├─────────────────────────────────────┤
│                                      │ 💬 피드백 입력: [여기에 수정 입력] [전송]│
└──────────────────────────────────────┴─────────────────────────────────────┘
```

- **특징**: 에이전트 간의 실시간 토론(SendMessage) 및 정합성 교정 과정을 시각화하여 사용자가 설계 루프를 능동적으로 제어할 수 있는 구조.
- **장점**: 기획 결함 발생 시 실시간으로 복구하는 에이전트 팀의 액션을 추적할 수 있어 투명성과 신뢰도가 극대화됨.
- **단점**: 콘솔 구조가 복잡하여 고도의 개발 및 테크니컬 기획에 숙련된 전문가 사용자층에 맞춰져 있음.
