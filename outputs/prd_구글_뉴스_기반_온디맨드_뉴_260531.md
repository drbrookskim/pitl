# [24.07.26] 온디맨드 맞춤형 뉴스 큐레이션 솔루션 PRD

## 0. 문서 개요
본 문서는 구글 뉴스 데이터를 기반으로 사용자의 특정 니즈에 맞춰 실시간으로 맞춤형 뉴스를 찾아주고, 핵심 내용을 요약 및 분석하여 정보 탐색 효율을 극대화하는 온디맨드 뉴스 큐레이션 서비스의 상세 요구사항을 정의합니다. 정보 과부하와 시간 부족으로 인해 필요한 정보를 선별하고 핵심을 파악하는 데 어려움을 겪는 바쁜 직장인, 전문가, 연구자, 투자자 등의 고객 문제를 해결하고, 기존 뉴스 서비스의 맞춤형 정보 부재 및 정보 신뢰성 문제를 개선하여 사용자 이탈 및 참여도 저하와 같은 비즈니스 문제를 해결하는 것을 목표로 합니다. AI/ML 기술을 활용한 개인화된 요약 및 분석 기능의 부재가 핵심 원인으로 진단되었으며, 이를 해결하기 위한 MVP(Phase 1) 및 고도화(Phase 2) 계획을 포함합니다.

## 1. 목표 (Goal)
본 프로젝트의 목표는 사용자의 정보 탐색 효율을 극대화하고 만족도를 높여 서비스의 지속적인 성장을 도모하는 것입니다.

| 항목 | 목표 수치 | 달성 기간 | 성공 기준 |
|---|---|---|---|
| **사용자 정보 탐색 시간 단축** | 30% 감소 | 3개월 내 | 사용자가 뉴스 탐색에 소요하는 평균 시간이 30% 감소함을 데이터로 확인 |
| **사용자 만족도 (NPS)** | 20점 증가 (예: 10점 -> 30점) | 6개월 내 | NPS 설문조사 결과가 20점 이상 증가함을 확인 |
| **주간 재방문율** | 20%p 증가 (예: 30% -> 50%) | 6개월 내 | 주간 활성 사용자 중 재방문율이 20%p 이상 증가함을 확인 |
| **유료 프리미엄 기능 전환율** | 3% 달성 | 12개월 내 (Phase 2 도입 후) | 유료 프리미엄 기능 출시 후 12개월 내 전환율 3% 달성 |

## 2. KPI 및 측정지표
프로젝트의 성공 여부를 판단하고 지속적으로 서비스를 개선하기 위한 핵심 성과 지표(KPI) 및 보조/모니터링 지표는 다음과 같습니다.

### 핵심 KPI 테이블
| 지표명 | 측정 방법 | 목표 수치 | 측정 주기 |
|---|---|---|---|
| **뉴스 탐색 시간 감소율** | (기존 평균 시간 - 서비스 이용 후 평균 시간) / 기존 평균 시간 | 55% 감소 (최종) | 월간 |
| **원하는 정보 탐색 성공률** | 사용자 설문조사 (원하는 정보를 찾았는지 여부) | 80% (최종) | 분기별 |
| **정보 과부하 인지 사용자 비율 감소율** | 사용자 설문조사 (정보 과부하 인지 여부) | 60% 감소 (최종) | 분기별 |
| **기존 뉴스 서비스 만족도 증가** | 사용자 설문조사 (5점 척도) | 4.2점 (최종) | 분기별 |
| **사용자 만족도 (NPS)** | 순추천고객지수 설문조사 | 20점 증가 | 분기별 |
| **주간 재방문율** | 주간 활성 사용자 중 7일 이내 재방문한 사용자 비율 | 20%p 증가 | 주간 |
| **유료 프리미엄 기능 전환율** | 유료 기능 이용 사용자 수 / 전체 활성 사용자 수 | 3% | 월간 (Phase 2 이후) |

### 보조 지표
*   **평균 세션 시간:** 사용자가 서비스에 머무는 평균 시간 (정보 소비 깊이 측정)
*   **페이지 뷰 수 (뉴스 상세/요약):** 사용자가 요약 또는 원문 뉴스를 얼마나 많이 조회하는지
*   **검색 기능 사용 빈도:** 온디맨드 검색 기능의 활용도
*   **관심 주제 설정 변경 빈도:** 개인화 기능 활용도
*   **피드백 제출률:** 사용자 피드백 시스템 참여도
*   **AI 요약/분석 요청 성공률:** AI 모델의 안정성 및 성능 지표
*   **Google News API 호출 성공률 및 응답 시간:** 외부 API 의존성 및 성능 지표

### 모니터링 지표
*   **서버 응답 시간:** 서비스 전반의 성능 지표
*   **오류 발생률:** 시스템 안정성 지표 (API 오류, AI 처리 오류 등)
*   **데이터 처리량:** 뉴스 수집 및 요약 처리량
*   **클라우드 리소스 사용량:** 비용 효율성 및 확장성 모니터링
*   **사용자 이탈률:** 특정 기능 사용 후 이탈하는 비율

## 4. 정책 정의

### 기본 정책
*   **실시간성:** Google News API를 통해 최신 뉴스 데이터를 실시간으로 수집 및 반영합니다.
*   **개인화:** 사용자의 관심 주제, 검색 기록, 피드백을 기반으로 맞춤형 뉴스 큐레이션을 제공합니다.
*   **AI 기반:** AI/ML 기술을 활용하여 뉴스 요약, 분석, 감성 분류 등을 수행합니다.
*   **투명성:** 요약된 뉴스의 원문 링크와 출처 매체 정보를 명확히 제공하여 정보의 신뢰성을 확보합니다.
*   **사용자 중심:** 사용자 피드백을 적극적으로 수집하고 서비스 개선에 반영합니다.
*   **데이터 보안 및 프라이버시:** 사용자 개인 정보 및 검색 기록은 관련 법규에 따라 안전하게 보호하고 관리합니다.

## 5. 서비스 구조 (유저 플로우 - Phase 1 MVP)

1.  **서비스 접속 및 로그인/회원가입:**
    *   사용자는 웹/앱을 통해 서비스에 접속합니다.
    *   기존 사용자는 로그인하고, 신규 사용자는 회원가입을 진행합니다. (소셜 로그인 포함)
2.  **관심 주제 설정:**
    *   로그인 후, 사용자는 초기 설정 또는 대시보드에서 관심 있는 주제(예: 기술, 경제, 정치, 특정 기업명)를 설정합니다.
    *   설정된 주제는 개인화된 뉴스 피드에 반영됩니다.
3.  **개인화된 대시보드 확인:**
    *   사용자는 설정된 관심 주제에 따라 큐레이션된 뉴스 피드를 대시보드에서 확인합니다.
    *   중요 뉴스 알림이 표시될 수 있습니다.
4.  **온디맨드 뉴스 검색 및 필터링:**
    *   사용자는 특정 키워드를 입력하여 뉴스를 검색합니다.
    *   기간, 매체, 지역, 감성 등 다차원 필터를 적용하여 검색 결과를 정교화합니다.
5.  **AI 기반 뉴스 요약 요청:**
    *   검색 결과 목록에서 관심 있는 뉴스 기사들을 선택하거나, 특정 검색 결과에 대해 AI 요약을 요청합니다.
    *   시스템은 선택된 복수의 기사를 AI로 분석하여 핵심 내용을 요약합니다.
6.  **요약된 뉴스 및 원문 확인:**
    *   사용자는 AI가 요약한 핵심 내용을 확인합니다.
    *   필요시 원문 링크를 통해 전체 기사를 읽을 수 있습니다.
7.  **사용자 피드백:**
    *   사용자는 요약 품질, 검색 결과의 관련성 등에 대해 피드백을 제출합니다.
    *   피드백은 AI 모델 개선 및 서비스 품질 향상에 활용됩니다.
8.  **로그아웃:**
    *   사용자는 서비스 이용을 마친 후 로그아웃합니다.

### 운영 요구사항
*   **모니터링:** 서비스 성능(응답 시간, 오류율), API 호출 현황, AI 모델 성능, 서버 리소스 사용량 등을 실시간으로 모니터링할 수 있는 시스템 구축.
*   **로깅:** 사용자 요청, 시스템 처리 과정, 오류 발생 시 상세 로그를 기록하고 관리하는 시스템 구축.
*   **장애 대응:** 주요 시스템 장애 발생 시 자동 알림 및 신속한 복구 프로세스 수립.
*   **성능:** 동시 사용자 N명 접속 시에도 안정적인 서비스 응답 시간(예: 3초 이내) 유지.
*   **확장성:** 사용자 증가 및 데이터 처리량 증가에 대비하여 시스템 확장이 용이하도록 설계.
*   **보안:** 사용자 데이터 및 API 키 등 민감 정보에 대한 암호화 및 접근 제어 정책 수립.

### 데이터 요구사항
*   **데이터 저장:** 수집된 뉴스 메타데이터, 요약 결과, 사용자 설정 정보, 피드백 등을 안정적으로 저장할 수 있는 데이터베이스 구축.
*   **데이터 프라이버시:** 사용자 개인 정보(이메일, 검색 기록 등)는 암호화하여 저장하고, 개인정보보호법 등 관련 법규를 준수.
*   **데이터 보존:** 뉴스 데이터는 N개월/N년 동안 보존하며, 사용자 요청에 따라 개인 정보는 즉시 삭제.
*   **데이터 백업:** 정기적인 데이터 백업 및 복구 시스템 구축.

## 7. UX/UI 고려사항
*   **로딩 처리:**
    *   뉴스 검색 및 AI 요약 처리 중에는 스켈레톤 UI 또는 로딩 스피너를 표시하여 사용자에게 진행 상황을 명확히 알립니다.
    *   긴 로딩이 예상될 경우, 예상 대기 시간 또는 "잠시만 기다려주세요"와 같은 안내 메시지를 제공합니다.
*   **실패 처리:**
    *   API 오류, AI 처리 실패 등 예외 상황 발생 시, 사용자에게 명확하고 이해하기 쉬운 오류 메시지를 제공합니다.
    *   가능한 경우, "다시 시도" 버튼이나 문제 해결을 위한 가이드(예: "검색어를 바꿔보세요")를 함께 제공합니다.
    *   고객센터 연결 또는 문의하기 기능을 쉽게 찾을 수 있도록 배치합니다.
*   **빈 화면 처리:**
    *   초기 접속 시, 관심 주제가 설정되지 않았거나 검색 기록이 없는 경우, "관심 주제를 설정하고 맞춤형 뉴스를 받아보세요!"와 같은 친절한 안내 메시지와 함께 설정 바로가기 버튼을 제공합니다.
    *   검색 결과가 없을 경우, "요청하신 키워드에 대한 뉴스를 찾을 수 없습니다. 다른 키워드로 검색해보시겠어요?"와 같은 메시지와 함께 관련성 높은 추천 키워드를 제시합니다.
*   **직관적인 인터페이스:**
    *   검색 필터는 사용자가 쉽게 접근하고 조작할 수 있도록 직관적인 UI를 제공합니다.
    *   뉴스 요약은 가독성이 높도록 명확한 레이아웃과 폰트를 사용합니다.
    *   피드백 제출 과정은 간결하고 부담 없이 참여할 수 있도록 설계합니다.

## 8. 기술 고려사항
*   **Google News API 통합:**
    *   API Rate Limit 및 할당량 관리를 위한 전략 수립 (캐싱, 호출 최적화).
    *   API 응답 오류 및 타임아웃에 대한 견고한 예외 처리 로직 구현.
    *   API 키 보안 관리 및 주기적인 갱신 정책 수립.
*   **AI/ML 모델 배포 및 관리:**
    *   뉴스 요약 및 감성 분석을 위한 NLP 모델 선정 및 학습 (BERT, GPT 등).
    *   모델 버전 관리 및 A/B 테스트를 통한 성능 개선 프로세스 구축.
    *   모델 추론(Inference) 속도 최적화 및 확장 가능한 배포 환경 (예: Kubernetes, Serverless Functions).
    *   AI 모델의 편향성 검증 및 지속적인 모니터링.
*   **데이터 파이프라인:**
    *   뉴스 데이터 수집, 전처리, 저장, 인덱싱을 위한 효율적인 데이터 파이프라인 구축 (예: Apache Kafka, Spark).
    *   대량의 텍스트 데이터를 처리하고 검색하기 위한 검색 엔진 (예: Elasticsearch) 활용 검토.
*   **확장 가능한 아키텍처:**
    *   클라우드 기반 (AWS, GCP, Azure) 마이크로서비스 아키텍처 설계로 유연한 확장성 확보.
    *   무상태(Stateless) 서비스 설계 및 로드 밸런싱을 통한 트래픽 분산.
*   **보안:**
    *   사용자 인증 및 권한 부여 (OAuth 2.0, JWT).
    *   데이터 전송 시 HTTPS 암호화 적용.
    *   SQL Injection, XSS 등 웹 취약점 방어.
    *   **데이터 입력 유효성 검사 및 필터링:** 사용자 입력(검색어, 피드백 등)에 대한 철저한 유효성 검사 및 악성 코드/스크립트 필터링을 통해 시스템 보안 강화.
    *   **보안 격리 환경:** AI 모델 실행 또는 민감한 데이터 처리 시, 컨테이너(Docker) 등을 활용한 격리된 환경에서 실행하여 잠재적 보안 위협으로부터 시스템 보호.

## 9. 실험 및 검증
*   **A/B 테스트:**
    *   다양한 요약 알고리즘, 검색 결과 정렬 방식, UI/UX 변경 사항에 대해 A/B 테스트를 진행하여 사용자 반응 및 지표 변화를 측정.
    *   예: 요약 문장 수, 감성 분석 표시 방식 등.
*   **사용자 인터뷰 및 설문조사:**
    *   MVP 출시 후 초기 사용자 그룹을 대상으로 심층 인터뷰를 진행하여 서비스 만족도, 개선점, 추가 니즈를 파악.
    *   정기적인 설문조사를 통해 NPS, 만족도, 정보 과부하 인지율 등 핵심 지표를 측정.
*   **데이터 기반 분석:**
    *   사용자 행동 데이터(검색 기록, 클릭률, 체류 시간 등)를 분석하여 기능 활용도 및 사용자 패턴을 이해하고 서비스 개선 방향 도출.
    *   AI 요약/분석의 정확도 및 편향성 지표를 지속적으로 모니터링하고 개선.
*   **피드백 시스템 활용:**
    *   사용자 피드백을 정기적으로 검토하고, 이를 바탕으로 기능 개선 및 버그 수정 우선순위를 결정.
