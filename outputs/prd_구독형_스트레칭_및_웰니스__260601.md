# 24.05.22 구독형 스트레칭 및 웰니스 모바일 서비스 PRD

## 0. 문서 개요
본 문서는 바쁜 현대인을 위한 맞춤형 10분 스트레칭 구독 서비스의 MVP 개발을 위한 상세 요구사항 명세서입니다. 사용자의 신체 상태를 진단하여 개인화된 루틴을 제공하고, 푸시 알림을 통해 습관 형성을 유도하여 기존 운동 앱의 낮은 리텐션 문제를 해결하는 것을 목적으로 합니다.

## 1. 목표 (Goal)
*   **프로젝트 목표:** 개인화된 스트레칭 루틴 제공 및 습관 형성 넛지 시스템 구축을 통한 시장 진입.
*   **성공 기준:**
    | 구분 | 목표 수치 |
    | :--- | :--- |
    | 가입자 수 | 런칭 3개월 내 50,000명 |
    | 유료 구독 전환율 | 8% |
    | 30일 리텐션 | 35% |

## 2. KPI 및 측정지표
*   **핵심 KPI:** 구독 전환율, 30일 리텐션, 일일 활성 사용자 수(DAU).
*   **보조 지표:** 루틴 완료율, 진단 테스트 이탈률, 푸시 알림 클릭률(CTR).
*   **모니터링 지표:** 결제 실패율, API 응답 시간(Latency), 앱 크래시율.

## 3. 사용자 및 이해관계자
*   **대상 사용자:** 2030 직장인, 재택근무자, 웰니스 입문자.
*   **제외 대상:** 고강도 운동을 선호하는 전문 운동인, 재활 치료가 시급한 환자(의료 서비스가 아님을 명시).
*   **이해관계자:** 프로덕트 팀(기획/디자인/개발), 콘텐츠 파트너(물리치료사/강사), 마케팅 팀.

## 4. 정책 정의
*   **상태값 정의:**
    | 상태 | 정의 |
    | :--- | :--- |
    | Free | 진단 테스트 및 샘플 영상만 이용 가능 |
    | Premium | 모든 루틴 및 개인화 콘텐츠 이용 가능 |
*   **예외 정책:**
    | 상황 | 대응 정책 |
    | :--- | :--- |
    | 결제 실패 | 결제 수단 변경 안내 및 3회 재시도 후 구독 일시 정지 |
    | API 타임아웃 | 3초 이상 지연 시 '네트워크 오류' 팝업 및 재시도 버튼 노출 |
    | 콘텐츠 재생 오류 | 캐시 삭제 안내 및 고객센터 문의 유도 |

## 5. 서비스 구조 (유저 플로우)
1. **온보딩:** 앱 설치 → 신체 상태 진단 테스트 → 결과 확인.
2. **메인:** 맞춤형 루틴 추천 → 루틴 시작.
3. **구독:** 결제 페이지 진입 → 7일 무료 체험 시작 → 구독 완료.
4. **습관:** 푸시 알림 수신 → 루틴 수행 → 완료 기록.

## 6. 상세 요구사항
| ID | 기능명 | 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 진단 테스트 | 신체 통증 부위/직업군 기반 설문 로직 | P0 |
| REQ-02 | 루틴 추천 | 진단 결과 기반 10분 영상 큐레이션 | P0 |
| REQ-03 | 구독 결제 | PG사 연동 및 7일 무료 체험 로직 | P0 |
| REQ-04 | 푸시 알림 | 사용자 설정 시간에 맞춘 루틴 알림 | P1 |
| REQ-05 | 마이페이지 | 운동 기록 및 구독 관리 | P1 |

*   **운영 요구사항:** 콘텐츠 관리자 페이지(CMS)를 통한 영상 업로드 및 태깅 기능.
*   **데이터 요구사항:** 사용자별 진단 데이터 및 루틴 완료 로그 수집(Amplitude/GA4 연동).

## 7. UX/UI 고려사항
*   **로딩:** 영상 로딩 시 스켈레톤 UI 적용.
*   **실패:** 네트워크 단절 시 오프라인 모드 안내 및 재연결 유도.
*   **빈 화면:** 루틴 기록이 없을 경우 '첫 루틴을 시작해보세요' CTA 버튼 배치.

## 8. 기술 고려사항
*   **보안:** 결제 정보는 PG사 서버에서 처리하며 앱 내 저장 금지.
*   **성능:** 영상 스트리밍 최적화(CDN 활용), API 응답 속도 500ms 이내 유지.

## 9. 실험 및 검증
*   **A/B 테스트:** 진단 테스트 문항 수에 따른 완료율 비교.
*   **검증:** 런칭 전 사내 베타 테스트를 통한 루틴 추천 알고리즘 정확도 검증.

## 10. 리스크 및 대응
*   **콘텐츠 차별화:** 전문 강사와의 독점 계약 및 퀄리티 검수 프로세스 강화.
*   **낮은 전환율:** 7일 무료 체험 및 첫 달 할인 프로모션으로 진입 장벽 완화.
*   **기술 오류:** 초기 영상 가이드 중심 운영으로 안정성 확보 후 고도화.
