반갑습니다. UX Architect 에이전트입니다. 'FlexFlow' 프로젝트의 핵심인 폼팩터 상태 감지(힌지 각도)를 기반으로, 사용자 경험의 목적성에 따른 3가지 타입의 ASCII 와이어프레임을 설계하였습니다.

---

### Type A: 현실/안정형 (Productivity-Focused)
**특징:** 비즈니스 프로페셔널을 위한 레이아웃입니다. 상단은 정보 확인, 하단은 제어 영역으로 명확히 분리하여 학습 비용을 최소화했습니다.

```text
┌──────────────────────────────────────┐
│ [상단: 뷰파인더/스크립트 영역]       │
│ ┌──────────────────────────────────┐ │
│ │ [이미지/텍스트 데이터]           │ │
│ └──────────────────────────────────┘ │
├──────────────────────────────────────┤
│ [하단: 제어/입력 영역]               │
│ ┌──────┐ ┌──────┐ ┌──────┐         │
│ │ 설정 │ │ 편집 │ │ 저장 │ [CTA]   │
│ └──────┘ └──────┘ └──────┘         │
└──────────────────────────────────────┘
```

---

### Type B: 디자인/감성몰입형 (Creative-Focused)
**특징:** 모바일 크리에이터를 위한 레이아웃입니다. 중앙 집중형 UI와 플로팅 컨트롤러를 배치하여 콘텐츠 몰입도를 극대화하고, 시각적 여백을 활용합니다.

```text
┌──────────────────────────────────────┐
│           [콘텐츠 메인 뷰]           │
│                                      │
│        ┌────────────────────┐        │
│        │   [미디어 영역]    │        │
│        └────────────────────┘        │
│                                      │
├──────────────────────────────────────┤
│ [플로팅 컨트롤러]                    │
│  ( < )  [  ||  ]  ( > )   [CTA]      │
└──────────────────────────────────────┘
```

---

### Type C: 미래형 동적 위젯 (Adaptive-Widget)
**특징:** 폴더블 폼팩터의 유연성을 강조한 레이아웃입니다. 힌지 각도에 따라 위젯의 크기와 위치가 유동적으로 변하며, 상태 변화를 시각화하는 인디케이터를 포함합니다.

```text
┌──────────────────────────────────────┐
│ [상태 인디케이터: 90도 거치 중...]    │
├──────────────────┬───────────────────┤
│ [위젯 A: 상태]   │ [위젯 B: 제어]    │
│ ┌──────────────┐ │ ┌───────────────┐ │
│ │ [그래프/수치]│ │ │ [슬라이더]    │ │
│ └──────────────┘ │ └───────────────┘ │
├──────────────────┴───────────────────┤
│ [공통 CTA: FlexFlow 활성화]          │
└──────────────────────────────────────┘
```

---

### UX Architect의 설계 의도 요약

1.  **Type A (안정형):** 업무 효율성을 중시하는 사용자에게 적합합니다. 물리적 버튼과 디지털 UI의 대응 관계를 직관적으로 설계하여 오작동을 방지합니다.
2.  **Type B (감성형):** 콘텐츠 제작 및 소비에 최적화되었습니다. 하단 영역을 최소화하여 시각적 방해 요소를 줄이고, 크리에이터의 창의적 활동을 지원합니다.
3.  **Type C (미래형):** 'FlexFlow'의 핵심인 센서 데이터를 UI로 시각화합니다. 사용자가 현재 기기의 상태를 인지하고, 시스템이 능동적으로 반응하고 있음을 체감하게 합니다.

**공통 적용 사항:**
*   **디바운스 대응:** 모든 타입은 1.5초의 상태 유지 후 UI가 전환되도록 설계되었습니다.
*   **CTA 배치:** 사용자의 엄지손가락 동선을 고려하여 하단 영역에 주요 액션 버튼을 배치하였습니다.
*   **확장성:** 서드파티 앱 미지원 시에도 위 레이아웃의 기본 틀을 유지하여 일관된 경험을 제공합니다.
