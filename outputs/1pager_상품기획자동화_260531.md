# [26.05.31] 유동적 상품기획 자동화 플랫폼 (PITL-Process) (Product-Planning Team)

## 1. 프로젝트 한 줄 요약
- 사용자가 입력한 러프한 아이디어 한 줄을 분석하여 철저한 정책 검증이 포함된 고품질 1-pager, PRD, ASCII 와이어프레임 3안을 단 1분 만에 자동 조화하여 배포하는 생성형 AI 상품 기획 플랫폼.

## 2. 대상
| 대상 | 상황/특징 |
| :--- | :--- |
| **주니어 서비스 기획자 / PM** | 아이디어를 스펙 문서로 구조화하는 시간(평균 12시간)이 부족하고, 철저한 예외 케이스(에러, 엣지 정책) 누락으로 개발 검토 단계에서 반려되는 빈도가 높음. |
| **초기 스타트업 창업가** | 빠른 MVP 검증이 필요하나 디자인 리소스가 부족하여 초기 레이아웃과 정보 구조(CTA)를 설계 및 시각화하는 데 난항을 겪음. |

## 3. 문제 정의
### 고객 문제
- **문서화 병목**: 한 줄의 유용한 직관이 개발 가능한 기능 요구사항(PRD) 문서로 완성되기까지의 물리적 리서치 및 문서 작성 리소스 과다 소요.
- **다양한 UI 시각화 부재**: UI/UX 시각화 시 다양한 설계안(안정형 vs 혁신형)을 비교하고 싶으나, 도구를 다루는 숙련도가 낮거나 디자인 피드백 루프가 느려 단일 시안으로 타협함.
- **예외 처리 누락**: 로딩, 네트워크 끊김, 데이터 공백 등 개발에서 필수적인 예외 케이스의 부재로 인해 기획서가 무한 루프 수정에 직면함.

### 비즈니스 문제
- **기획 단계 지연**: 기획서 작성 및 합의 지연이 전체 스프린트 주기(2주)의 병목(평균 40% 이상 차지)으로 작용하여 신기능 시장 타임투마켓(Time-to-Market) 지연.
- **소통 오버헤드**: PM-디자이너-개발자 간의 레이아웃 해석 차이로 불필요한 재작업 및 미팅 공수 급증.

### 데이터 기반 현황
| 주요 메트릭 | 현재 수치 | 인사이트 |
| :--- | :--- | :--- |
| **스펙 작성 평균 공수** | 14.5시간 | 단순 구조화 및 템플릿 채우기 작업에 60% 이상의 공수 낭비 |
| **기획서 개발 반려율** | 38.2% | 예외 케이스(로딩/실패 상태) 정의 누락 및 정책 모호성이 주원인 |
| **초기 UI 합의 변경 횟수** | 4.2회 | 화면 구조가 시각화되지 않아 텍스트 해석의 차이로 수시 변동 발생 |

### Root Cause
- **Case 1**: 표준 PRD 템플릿의 파편화 및 작성자의 엣지 케이스 시나리오 인지 부족.
- **Case 2**: Figma 등 전문 UI 툴 활용의 높은 진입장벽과 아이디어를 레이아웃으로 변환하는 추상화 병목.

## 4. 목표
| 항목 | 목표 수치 |
| :--- | :--- |
| **기획 초안 도출 시간** | 14.5시간 → **1분 이내** (99.8% 단축) |
| **개발 검토 통과율** | 61.8% → **90% 이상** (예외 정책 자동 탑재) |
| **시안 다각화** | 1개 단순 안 → **3가지 상이한 가치 지향 시안(Type A/B/C) 필수 제공** |

## 5. 현황 및 분석
- **현장 실무 인터뷰**: "Figma로 그리기는 리소스가 크고 텍스트로만 쓰자니 디자이너와 개발자가 딴소리를 합니다. 1-pager의 방향성과 PRD 요구사항, 그리고 화면의 기본 뼈대가 동시에 매핑되는 스피디한 기획 오케스트레이터가 절실합니다."
- **시장 동향**: Notion AI나 ChatGPT는 단순 텍스트 PRD만 나열할 뿐, 구체적인 예외 정책 설계나 화면 레이아웃(ASCII)과의 삼각 정합성 검증이 유기적으로 엮이지 않아 실무 도입에 한계가 있음.

## 6. 해결방안 (Phase 단위 기획)
- **Phase 1 (Core CLI Engine)**: 사용자의 텍스트 아이디어를 파싱하여 삼각 검증이 완료된 1-pager, PRD, ASCII 와이어프레임 3안 마크다운 문서를 배포하는 오케스트레이션 엔진 구축.
- **Phase 2 (Visual Web GUI)**: 마크다운 및 ASCII 기반의 결과물을 세련된 블록 형태와 글쓰기 캔버스로 편집할 수 있으며, 클릭 한 번으로 PDF 및 피그마 초안 파일로 연동 내보내기하는 프리미엄 웹 대시보드 확장.

## 7. 리스크
| 영향도 | 리스크 | 대응 방안 |

## 8. 중요도 및 긴급도
- **중요도**: **상** (제품 출시의 가장 큰 지연 요소인 기획 병목을 제거하여 신제품 순환 주기를 비약적으로 단축함)
- **긴급도**: **상** (AI 코딩 어시스턴트 및 생산성 도구가 급변하는 2026년 현재, 기획 단계의 자동화는 시장 선점을 위한 최우선 화두임)
