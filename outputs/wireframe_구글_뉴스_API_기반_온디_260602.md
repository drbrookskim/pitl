## [공통] 화면 목적 및 공통 CTA 정의

*   **화면 목적:** 사용자가 등록한 관심 키워드 기반의 정제된 뉴스(중복 제거 및 AI 3줄 요약)를 인지적 부하 없이 신속하게 소비하고, 외부 채널 연동 및 유료 구독(Premium)으로의 전환을 유도함.
*   **공통 핵심 CTA (Call to Action):**
    1.  **[+ 키워드 등록/추가]**: 개인화된 뉴스 수집의 시작점.
    2.  **[Premium 업그레이드 ⚡]**: 무제한 키워드, AI 요약, 외부 발송 잠금 해제.
    3.  **[원문 읽기 ↗]**: 저작권 준수를 위한 아웃링크 이동.
    4.  **[★ 북마크/저장]**: 중요 정보 아카이빙.
    5.  **[👍 / 👎]**: AI 요약 품질 피드백 루프 형성.

### 2. 레이아웃 특징 및 정보 구조
*   **2단 분할 그리드 구조 (Left Sidebar + Main Content):** 전통적인 SaaS 및 포털 대시보드 레이아웃을 채택하여 사용자의 학습 비용을 최소화합니다.
*   **좌측 네비게이션 영역:** 현재 등록된 키워드 현황(무료 플랜의 3개 제한 직관적 표시)과 외부 채널 연동 상태를 한눈에 파악할 수 있습니다.
*   **우측 메인 피드 영역:** 필터링 도구와 뉴스 카드가 수직으로 나열되어 대량의 정보를 안정적이고 체계적으로 훑어볼 수 있습니다.

### 3. 타겟 고객 및 UX 설계 의도
*   **타겟:** 보수적인 비즈니스 종사자, 기존 포털 뉴스 레이아웃에 익숙한 30~50대 전문직.
*   **설계 의도:** 정보의 '통제권'이 사용자에게 있음을 시각적으로 강조합니다. 좌측의 키워드 관리와 우측의 피드가 유기적으로 연결되어 작동하며, 예외 상황(결제 만료 경고 등)을 상단 배너와 좌측 알림창을 통해 명확히 인지시킵니다.

*   **감성적 타이포그래피 및 여백:** 기사의 핵심 인용구("더 가볍고, 더 빠르며...")를 카드 중앙에 배치하여 감성적인 몰입감을 제공합니다.

*   **타겟:** 정보 피로감을 극도로 느끼는 20~30대 트렌드 세터, 미니멀한 디자인과 킨포크 스타일의 텍스트 소비를 선호하는 사용자.
*   **설계 의도:** '하루에 단 몇 개의 고품질 뉴스만 깊게 읽는다'는 느낌을 전달합니다. 좌우 슬라이드(Swipe) 인터랙션을 유도하여 한 번에 하나의 정보만 처리하게 함으로써 인지적 과부하를 원천적으로 차단합니다.

## Type C: 미래형 동적 위젯 (Command-Line / Modular Widget)

### 1. ASCII 와이어프레임
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ >_ BRIEFLY_OS [v1.0]                                    [SYS_STATUS: ACTIVE] │
├──────────────────────────────────────────────────────────────────────────────┤
│ > [ Command: 키워드 추가는 '/add', 외부 연동은 '/connect'를 입력하세요     ] │
├──────────────────────────────────────┬───────────────────────────────────────┤
│ [W1: 실시간 모니터링 키워드]         │ [W2: 실시간 뉴스 스트리밍 피드]       │
│ ┌──────────────────────────────────┐ │ ┌───────────────────────────────────┐ │
│ │ #인공지능_AI  [ON] [Slack]       │ │ │ [14:02] #인공지능_AI              │ │
│ │ #SaaS_비즈   [ON] [Email]        │ │ │ > OpenAI GPT-4o-mini 출시 완료    │ │
│ │ #핀테크      [OFF]               │ │ │   [AI 요약 보기]   [원문 링크 ↗]   │ │
│ │                                  │ │ │ ───────────────────────────────── │ │
│ │ [⚡ Premium: 무제한 키워드 활성화]│ │ │ [13:45] #SaaS_비즈                │ │

│ [W3: 시스템 성능 및 비용 절감 지표]  │ └───────────────────────────────────┘ │
│ ┌──────────────────────────────────┐ │ [W4: 퀵 액션 콘솔]                    │
│ │ • Redis Cache Hit Rate : 72.4%   │ │ ┌───────────────────────────────────┐ │

│ │ • 오늘 절약한 정보 탐색: 32분    │ │ │ [🔌 외부 채널(Slack/Email) 연동]  │ │
│ └──────────────────────────────────┘ │ └───────────────────────────────────┘ │
└──────────────────────────────────────┴───────────────────────────────────────┘
```

### 2. 레이아웃 특징 및 정보 구조
*   **커맨드 라인 인터페이스(CLI) + 모듈러 위젯 구조:** 개발자 도구 또는 금융 트레이딩 터미널을 연상시키는 미래지향적 그리드 레이아웃입니다.
*   **커맨드 바 (Command Bar):** 화면 상단에 검색창 대신 명령어를 입력할 수 있는 통합 입력창(`> [ Command: ... ]`)을 배치하여 키보드 중심의 빠른 조작을 지원합니다.
*   **독립적 위젯 구성 (W1~W4):** 키워드 상태, 실시간 스트리밍 피드, 시스템 메트릭(Redis 캐시율, 절약 시간), 퀵 액션이 각각 독립된 위젯 카드로 분할되어 실시간 대시보드 역할을 수행합니다.

### 3. 타겟 고객 및 UX 설계 의도
*   **타겟:** IT 업계 개발자, 엔지니어, 테크 스타트업 창업자, 데이터 분석가 등 생산성 극대화를 추구하는 파워 유저.
*   **설계 의도:** 정보를 '수집하고 제어하는 시스템'을 직접 운영하는 듯한 희열을 제공합니다. 특히 **[W3: 시스템 성능 및 비용 절약 지표]** 위젯을 통해 서비스의 기술적 신뢰도(Redis 캐시 효율, 절약된 시간)를 시각적으로 증명하여 유료 구독의 당위성을 기술적으로 설득합니다.

## 3가지 시안 비교 요약

| 비교 항목 | Type A: 현실/안정형 | Type B: 디자인/감성몰입형 | Type C: 미래형 동적 위젯 |
| :--- | :--- | :--- | :--- |
| **정보 밀도** | 높음 (전형적인 대시보드) | 낮음 (미니멀리즘 카드) | 매우 높음 (실시간 위젯) |
| **인지적 부하** | 보통 (익숙한 구조) | 매우 낮음 (하나씩 집중) | 높음 (다양한 데이터 동시 노출) |
| **주요 인터랙션** | 마우스 클릭, 스크롤 | 스와이프(Swipe), 페이지네이션 | 키보드 단축키, 커맨드 입력 |
| **구독 유도 전략** | 무료 제한(3개) 및 경고 배너 | 프리미엄 전용 기능의 감성적 가치 강조 | 시스템 효율성 및 생산성 지표 시각화 |
