# [24.05.20] 정보 과부하 해결을 위한 온디맨드 뉴스 큐레이션 플랫폼 'Briefly' 상세 요건 명세서 (PRD)

---

## 0. 문서 개요

### 0.1 프로젝트 요약
본 프로젝트는 사용자가 지정한 관심 키워드 및 주제에 맞춰 실시간으로 뉴스를 수집(구글 뉴스 API 활용)하고, LLM(대형 언어 모델) 기반 AI 기술을 통해 중복 기사 필터링 및 3줄 요약을 제공하는 온디맨드 뉴스 큐레이션 서비스 **'Briefly'**의 개발을 목표로 합니다. 수집 및 요약된 정보는 웹 뷰어뿐만 아니라 슬랙(Slack), 이메일, 카카오톡 등 사용자가 원하는 외부 채널로 자동 배달되어 정보 탐색 시간을 획기적으로 단축시킵니다.

### 0.2 배경 및 필요성
현대 비즈니스 종사자 및 정보 소비자는 포털 사이트의 광고성 기사, 중복 기사, 자극적인 황색 언론으로 인해 심각한 '정보 과부하(Information Overload)' 상태에 직면해 있습니다. 조사에 따르면 일평균 뉴스 검색 및 필터링에 소모되는 시간은 42분에 달합니다. 'Briefly'는 공급자 중심의 대량 송출 방식을 탈피하여, 철저히 사용자 중심의 '온디맨드 필터링 및 요약 기술'을 통해 이 문제를 해결하고 유료 구독 모델(Premium Plan)을 통해 비즈니스 지속 가능성을 확보하고자 합니다.

## 1. 목표 (Goal)

### 1.1 프로젝트 목표
* **사용자 가치 혁신:** 무분별한 뉴스 탐색 시간을 줄이고, 신뢰할 수 있는 핵심 정보만 빠르게 습득할 수 있는 환경 제공.
* **비즈니스 모델 검증:** 개인화된 알림 및 AI 요약 가치를 바탕으로 유료 구독 전환율 5.0% 달성.
* **기술적 효율성 확보:** Redis 캐싱 및 클러스터링 알고리즘을 통한 API 비용 최적화 및 시스템 안정성 확보.

### 1.2 성공 기준 (Success Criteria)
| 목표 항목 | 현재 수치 (As-Is) | 목표 수치 (To-Be) | 측정 방법 및 검증 도구 |
| :--- | :--- | :--- | :--- |
| **뉴스 탐색 시간 단축** | 일평균 42분 | **10분 이하 (75% 감소)** | 사용자 대상 정기 설문조사 및 인앱 체류 시간 분석 |
| **서비스 리텐션 (D+7)** | 15% | **45% 이상** | Amplitude / Mixpanel 코호트 분석 (가입 후 7일째 재방문율) |
| **유료 구독 전환율** | 1.2% | **5.0% 이상** | Stripe / PG사 결제 완료 데이터 기반 (가입자 대비 유료 결제자 비율) |
| **API 호출 비용 절감** | - | **기존 대비 30% 절감** | AWS CloudWatch 및 Google API Console 비용 모니터링 |

## 2. KPI 및 측정지표

### 2.1 핵심 KPI (Core KPIs)
* **D+7 Retention Rate:** 가입 후 7일 차에 서비스를 다시 이용(웹 로그인 또는 외부 알림 클릭)한 사용자의 비율. (목표: 45%)
* **Paid Subscription Conversion Rate:** 전체 가입자 중 Premium Plan(유료) 결제 유저의 비율. (목표: 5.0%)
* **LTV (Customer Lifetime Value) 대 CAC (Customer Acquisition Cost) 비율:** 마케팅 효율성 검증 지표. (목표: LTV/CAC > 3.0)

### 2.2 보조 지표 (Secondary Metrics)
* **인당 평균 등록 키워드 수:** 유저가 등록하여 모니터링하는 활성 키워드의 평균 개수 (목표: 3.5개 이상).
* **알림 열람률 (CTR):** 슬랙, 이메일, 카카오톡으로 발송된 뉴스 요약본의 링크 클릭률 (목표: 35% 이상).
* **AI 요약 만족도 피드백:** 요약 하단의 추천/비추천(Thumbs Up/Down) 중 추천 비율 (목표: 90% 이상).

### 2.3 시스템 및 모니터링 지표 (System Metrics)
* **Redis Cache Hit Rate:** 구글 뉴스 API 호출 시 캐시를 통해 처리되는 비율 (목표: 70% 이상).
* **LLM API Latency & Timeout Rate:** AI 요약 생성 시 평균 지연 시간(목표: 3초 이하) 및 타임아웃 발생률(목표: 0.5% 이하).
* **결제 성공률 (PG Success Rate):** 결제 시도 대비 최종 승인 완료 비율 (목표: 99% 이상).

* **전문직 및 비즈니스 종사자 (기획자, 마케터, 투자자 등):** 매일 특정 산업 트렌드 및 경쟁사 동향을 모니터링해야 하는 직군.
* **대학생 및 연구원:** 특정 학술 주제, 기술 트렌드, 시사 이슈에 대한 깊이 있는 뉴스를 아카이빙하고자 하는 사용자.
* **정보 피로감을 느끼는 대중:** 포털의 자극적인 기사 및 중복 기사에 지쳐 정제된 뉴스만 읽고 싶어 하는 사용자.

## 4. 정책 정의

### 4.1 기본 정책
* **요금제 정책:**
  * **Free Plan (무료):** 최대 3개 키워드 등록 가능, 웹 뷰어에서만 뉴스 조회 가능 (AI 요약 미제공, 외부 채널 알림 미제공).
  * **Premium Plan (유료 - 월 $9.99):** 키워드 등록 무제한, AI 3줄 요약 제공, 외부 채널(슬랙, 이메일, 카카오톡) 실시간/배치 알림 제공.
* **저작권 준수 정책:**
  * 모든 뉴스는 원문 링크 아웃(Outlink) 방식을 취하며, 인앱 브라우저 또는 새 창으로 열려야 함.
  * AI 요약본은 원문의 단순 요약 형태로만 제공하며, 원문 기사 전체를 무단 전재하지 않음(공정 이용 범위 준수).

## 7. UX/UI 고려사항

### 7.1 로딩 상태 (Loading States)
* **스켈레톤 UI 적용:** 뉴스 피드 로딩 시, 텍스트와 이미지가 들어갈 자리에 부드러운 애니메이션이 적용된 스켈레톤(Skeleton) 화면을 노출하여 체감 대기 시간을 단축시킵니다.
* **AI 요약 프로그레스 바:** 사용자가 실시간 요약을 요청할 경우, "AI가 기사를 분석하고 요약하는 중입니다 (약 3초 소모)"라는 안내와 함께 마이크로 인터랙션(로딩 스피너)을 제공합니다.

### 7.2 실패 상태 (Failure States)
* **네트워크 오류/API 타임아웃:** "인터넷 연결이 불안정하거나 뉴스 소스를 불러오지 못했습니다."라는 메시지와 함께 **[다시 시도]** 버튼을 직관적으로 배치합니다.
* **결제 실패 알림:** 대시보드 상단에 주황색 경고 배너를 노출하여 "결제 카드가 만료되었거나 잔액이 부족합니다. 5일 후 무료 플랜으로 전환됩니다. [결제 정보 업데이트]" 문구를 제공합니다.

### 7.3 빈 화면 처리 (Empty States)
* **신규 유저/키워드 미등록 상태:** "아직 등록된 키워드가 없습니다. 관심 있는 키워드를 등록하고 3초 만에 요약된 뉴스를 받아보세요!"라는 문구와 함께 **[첫 키워드 등록하기]** CTA(Call to Action) 버튼을 화면 중앙에 크게 배치합니다.
* **검색 결과 없음:** "오늘 해당 키워드에 대한 새로운 뉴스가 없습니다. 키워드를 조금 더 넓은 범위로 변경해보는 건 어떨까요? (예: 'ChatGPT' -> '인공지능')"과 같은 가이드를 제공합니다.

## 8. 기술 고려사항

### 8.1 API 캐싱 및 비용 최적화 (Redis)
* **캐시 키 설계:** `news:keyword:{keyword_hash}:date` 구조로 캐시 키를 생성합니다.
* **TTL(Time-To-Live):** 2시간으로 설정하여, 동일 키워드에 대해 여러 사용자가 조회하더라도 구글 뉴스 API 호출은 2시간에 1회만 발생하도록 제한합니다.

### 8.2 보안 및 격리 (Security & Isolation)
* **API Key 관리:** Google News API, OpenAI API, PG사 Secret Key 등은 AWS Secrets Manager 또는 환경 변수(`.env`)를 통해 엄격히 격리하여 관리하며, 클라이언트 코드에 절대 노출되지 않도록 합니다.
* **Rate Limiting:** 악의적인 디도스(DDoS) 공격 및 API 남용을 방지하기 위해, IP당/계정당 분당 최대 호출 횟수(Rate Limit)를 설정합니다 (예: 분당 최대 60회 요청 제한).

### 8.3 아웃링크 및 보안 정책
* **보안 헤더 적용:** 뉴스 원문 링크 이동 시 `target="_blank" rel="noopener noreferrer"` 속성을 강제 적용하여 부모 페이지의 제어권을 보호하고 피싱 사이트 리스크를 원천 차단합니다.

### 8.4 LLM 프롬프트 인젝션 방지 (Prompt Injection Defense)
* 사용자가 등록한 키워드에 시스템 명령어를 주입하여 LLM의 시스템 프롬프트를 탈취하려는 시도를 방지하기 위해, 입력값에 대한 필터링(특수문자 제한 및 명령어 이스케이프 처리)을 적용합니다.

## 9. 실험 및 검증

### 9.1 A/B 테스트 시나리오
* **실험 1: AI 요약 레이아웃 테스트**
  * **대조군 (A):** 단순 3줄 줄글 형태의 요약 노출.
  * **실험군 (B):** 핵심 키워드 태그 3개 + 이모지를 활용한 Bullet Point 3줄 요약 노출.
  * **측정 지표:** 요약 만족도 피드백(Thumbs Up) 비율 및 인앱 체류 시간.
* **실험 2: 온보딩 단계 구독 유도 팝업**
  * **대조군 (A):** 가입 완료 후 바로 무료 대시보드로 이동.
  * **실험군 (B):** 가입 완료 직후 "Premium Plan 7일 무료 체험" 혜택을 강조하는 팝업 노출 후 카드 등록 유도.
  * **측정 지표:** 유료 구독 전환율(Conversion Rate).

### 9.2 사용자 피드백 루프 (Feedback Loop)
* **정량 피드백:** 모든 AI 요약 카드 하단에 "이 요약이 도움이 되었나요? 👍 👎" 버튼을 배치하여 실시간으로 데이터 수집.
* **정성 피드백:** 가입 후 14일이 경과한 활성 사용자 중 무작위 500명을 대상으로 인앱 NPS(Net Promoter Score) 조사 팝업 노출.
