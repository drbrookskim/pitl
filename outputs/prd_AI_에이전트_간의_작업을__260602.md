# [24.10.24] Multi-Agent Orchestration: AI 에이전트 협업 모니터링 대시보드 'AgentFlow'

## 0. 문서 개요
본 문서는 멀티 에이전트 시스템의 협업 과정을 실시간으로 시각화하고, 비정상 루프 차단 및 인간 개입(Human-in-the-loop)을 지원하는 'AgentFlow' 대시보드 구축을 위한 상세 요구사항 명세서입니다.

## 1. 목표 (Goal)
* **목표:** 멀티 에이전트 협업의 가시성 확보 및 운영 효율 극대화
* **성공 기준:**
| 항목 | 목표 수치 | 측정 방법 |
| :--- | :--- | :--- |
| 에이전트 오류 감지 시간 | 5분 이내 | 알림 발생 시점 ~ 운영자 확인 시점 |
| 멀티 에이전트 성공률 | 92% 이상 | 전체 태스크 대비 완료(Done) 비율 |
| 디버깅 소요 시간 | 주당 3시간 이하 | 개발팀 티켓 처리 시간 합산 |
| API 비용 낭비율 | 90% 절감 | 루프 차단 알고리즘 작동 횟수 기반 산출 |

## 2. KPI 및 측정지표
* **핵심 KPI:**
    * **MTTD (Mean Time to Detect):** 에이전트 장애 감지 평균 시간
    * **Task Success Rate:** 최종 완료된 태스크의 비율
    * **Cost Efficiency Ratio:** 전체 API 비용 대비 루프 차단으로 절감된 비용
* **보조 지표:** 에이전트별 평균 응답 속도, 인간 개입(Approval) 승인 소요 시간
* **모니터링 지표:** 실시간 동시 접속 에이전트 수, WebSocket 연결 안정성

## 3. 사용자 및 이해관계자
| 대상 | 역할 | 니즈 |
| :--- | :--- | :--- |
| AI Ops 담당자 | 운영 및 관리 | 실시간 상태 모니터링 및 장애 대응 |
| AI 엔지니어 | 개발 및 디버깅 | 에이전트 간 통신 로그 및 루프 추적 |
| 비즈니스 PM | ROI 관리 | API 비용 모니터링 및 승인 프로세스 제어 |

## 4. 정책 정의
* **상태값 정의:**
    * `QUEUED`: 태스크 대기 중
    * `IN_PROGRESS`: 에이전트 작업 수행 중
    * `HUMAN_APPROVAL`: 인간 개입 대기 중
    * `COMPLETED`: 작업 완료
    * `FAILED`: 에러 발생
    * `BLOCKED`: 루프 감지로 인한 자동 차단
* **예외 처리 정책:**
    * **API 타임아웃:** 3회 재시도 후 `FAILED` 처리 및 관리자 알림.
    * **무한 루프:** 동일 태스크 3회 반복 시 즉시 `BLOCKED` 처리 및 프로세스 일시정지.
    * **결제 실패:** API 크레딧 부족 시 모든 에이전트 작업 즉시 일시정지 및 결제 페이지 리다이렉트.

## 5. 서비스 구조 (유저 플로우)
1. **연동:** SDK를 통해 에이전트 워크플로우를 AgentFlow 서버에 등록.
2. **모니터링:** 칸반 보드에서 실시간 상태 확인.
3. **개입:** `HUMAN_APPROVAL` 상태 카드 발생 시 운영자가 내용 확인 후 '승인' 또는 '반려' 수행.
4. **대응:** `BLOCKED` 카드 발생 시 로그 확인 후 프롬프트 수정 및 재개.

## 6. 상세 요구사항
| ID | 기능명 | 우선순위 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 칸반 보드 시각화 | P0 | 에이전트 상태별 카드 배치 및 실시간 업데이트 |
| REQ-02 | 실시간 알림 | P0 | 슬랙/이메일 연동, 장애 발생 시 즉시 알림 |
| REQ-03 | 루프 차단 엔진 | P0 | 동일 태스크 반복 감지 시 자동 차단 |
| REQ-04 | 인간 개입(Approval) | P1 | 승인/반려 버튼 및 수정 기능 제공 |
| REQ-05 | 비용 분석 대시보드 | P2 | 에이전트별 토큰 소모량 및 비용 시각화 |

## 7. UX/UI 고려사항
* **로딩:** 데이터 동기화 중 스켈레톤 UI 적용.
* **실패:** API 연결 끊김 시 '재연결 시도 중' 토스트 메시지 노출.
* **빈 화면:** 에이전트 연동 전 'SDK 연동 가이드' 페이지로 유도.

## 8. 기술 고려사항
* **데이터 수집:** 비동기 이벤트 리스너(WebSocket)를 사용하여 메인 에이전트 로직에 부하 최소화.
* **보안:** API 키 암호화 저장 및 역할 기반 접근 제어(RBAC) 적용.
* **호환성:** OpenTelemetry 표준을 준수하여 다양한 프레임워크(CrewAI, AutoGen 등) 지원.

## 9. 실험 및 검증
* **A/B 테스트:** 루프 차단 알고리즘 적용 전후의 API 비용 절감 폭 측정.
* **사용성 테스트:** AI Ops 담당자 대상 칸반 보드 조작 편의성 인터뷰(UT).

## 10. 리스크 및 대응
* **리스크:** 프레임워크별 데이터 포맷 상이.
* **대응:** 표준화된 Webhook API 및 SDK 제공을 통해 데이터 정규화 수행.
