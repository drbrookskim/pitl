# [24.10.24] AI 기반 맞춤형 웰니스 서비스 'FlexFlow' PRD

## 0. 문서 개요
* **프로젝트 요약:** AI 자세 분석 기술을 활용하여 사용자 체형에 최적화된 10분 스트레칭 루틴을 제공하는 구독형 모바일 서비스.
* **배경:** 현대인의 만성 근골격계 통증 해결을 위해 고가의 오프라인 케어를 대체하고, 유튜브의 일방향적 한계를 극복하는 '마이크로 웰니스' 솔루션 제공.

## 1. 목표 (Goal)
* **프로젝트 목표:** 6개월 내 10만 다운로드 달성 및 유료 구독 전환율 8.5% 확보.
* **성공 기준:**
| 구분 | 목표 수치 | 측정 기준 |
| :--- | :--- | :--- |
| 유료 구독 전환율 | 8.5% | 7일 무료 체험 종료 후 유료 결제 전환 |
| D+30 리텐션 | 65.0% | 가입 30일 후 앱 재방문율 |
| NPS | 75점 이상 | 분기별 사용자 만족도 조사 |

## 2. KPI 및 측정지표
* **핵심 KPI:** 유료 구독 전환율, D+30 리텐션, 일일 활성 사용자 수(DAU).
* **보조 지표:** 루틴 완료율(Completion Rate), AI 진단 수행률, 푸시 알림 클릭률(CTR).
* **모니터링 지표:** API 응답 시간, 결제 실패율, 앱 크래시율.

## 3. 사용자 및 이해관계자
* **대상 사용자:** 2040 사무직 직장인, 홈트레이닝 입문자.
* **이해관계자:**
| 이해관계자 | 역할 |
| :--- | :--- |
| 제품팀 | 서비스 기획 및 로드맵 관리 |
| AI 엔지니어 | 자세 분석 모델 개발 및 고도화 |
| 콘텐츠팀 | 스트레칭 영상 제작 및 전문의 검수 |
| 운영팀 | 고객 문의 대응 및 B2B 제휴 관리 |

## 4. 정책 정의
* **기본 정책:** 7일 무료 체험 후 자동 구독 전환(사전 고지 필수).
* **상태값 정의:**
| 상태값 | 설명 |
| :--- | :--- |
| Active | 구독 중인 상태 |
| Trial | 7일 무료 체험 기간 |
| Churn | 구독 해지 및 만료 상태 |
| Pending | 결제 대기 중 |
* **예외 정책:**
| 상황 | 정책 |
| :--- | :--- |
| 결제 실패 | 3회 재시도 후 구독 일시 정지 및 알림 발송 |
| API 타임아웃 | 5초 초과 시 '네트워크 오류' 팝업 및 재시도 버튼 노출 |
| AI 진단 실패 | 조도 부족/전신 미인식 시 가이드 문구 노출 및 재촬영 유도 |

## 5. 서비스 구조 (User Flow)
1. **온보딩:** 회원가입 → 통증 부위/직업군 설문.
2. **진단:** AI 카메라 촬영 → 체형 분석 리포트 생성.
3. **루틴:** 맞춤형 10분 루틴 추천 → 스트레칭 수행.
4. **습관:** 푸시 알림 → 루틴 완료 → 스트릭(Streak) 적립.

## 6. 상세 요구사항
| ID | 기능명 | 우선순위 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| REQ-01 | AI 자세 진단 | P0 | 스마트폰 카메라를 통한 거북목/골반 비대칭 분석 |
| REQ-02 | 맞춤형 루틴 | P0 | 진단 결과 기반 10분 스트레칭 영상 큐레이션 |
| REQ-03 | 구독 결제 | P0 | 인앱 결제(IAP) 및 정기 구독 관리 |
| REQ-04 | 푸시 알림 | P1 | 사용자 설정 시간에 맞춘 루틴 수행 알림 |
| REQ-05 | 스트릭 관리 | P2 | 연속 달성 일수 시각화 및 리워드 연동 |

* **운영 요구사항:** 관리자 대시보드(사용자 통계, 콘텐츠 관리, 결제 현황).
* **데이터 요구사항:** 개인정보 비식별화 처리, AI 학습용 영상 데이터 암호화 저장.

## 7. UX/UI 고려사항
* **로딩:** AI 분석 중 '분석 중입니다' 애니메이션 및 팁 문구 노출.
* **실패:** 네트워크 오류 시 오프라인 모드(저장된 루틴) 제공 가능 여부 검토.
* **빈 화면:** 루틴 완료 전 '오늘의 루틴을 시작해보세요' CTA 버튼 배치.

## 8. 기술 고려사항
* **보안:** AI 분석 영상은 서버 저장 없이 실시간 분석 후 즉시 파기(Privacy-by-design).
* **성능:** 영상 스트리밍 최적화(CDN 활용), API 응답 시간 2초 이내 최적화.

## 9. 실험 및 검증
* **A/B 테스트:** 푸시 알림 문구(감성형 vs 정보형)에 따른 클릭률 비교.
* **사용성 테스트:** MVP 출시 전 50명 대상 FGI(Focus Group Interview) 진행.

## 10. 리스크 및 대응
* **리스크:** 스트레칭 중 부상 발생.
* **대응:** 영상 내 '주의사항' 자막 필수 삽입, 전문의 검수 인증 마크 노출, 면책 조항 동의 절차 강화.
