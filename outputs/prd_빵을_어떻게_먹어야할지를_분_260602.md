# [24.10.24] 맞춤형 빵 소비 가이드 플랫폼 'Bread-it' PRD

## 0. 문서 개요
본 문서는 빵 구매 후 보관 및 조리 과정에서 발생하는 사용자 경험 저하를 해결하기 위한 'Bread-it' 서비스의 상세 요구사항을 정의합니다. 빵의 노화 메커니즘을 고려한 최적의 가이드를 제공하여 빵 폐기율을 낮추고, 홈카페 경험을 극대화하는 것을 목적으로 합니다.

## 1. 목표 (Goal)
* **프로젝트 목표:** 빵 종류별/가전별 최적화된 조리/보관 가이드를 제공하여 빵 소비 경험의 표준화 및 폐기율 감소.
* **성공 기준:**
| 항목 | 목표 수치 |
| :--- | :--- |
| MAU | 100,000명 (6개월 내) |
| 사용자 기준 빵 폐기율 | 기존 대비 50% 감소 |
| 가이드 저장 및 공유율 | 방문자 대비 35% 이상 |

## 2. KPI 및 측정지표
| 구분 | 지표명 | 측정 방법 |
| :--- | :--- | :--- |
| **핵심 KPI** | 빵 폐기 감소율 | 설문조사 및 앱 내 '소비 완료' 기록 데이터 분석 |
| **보조 지표** | 가이드 활용률 | 빵 검색 후 타이머 실행 완료율 |
| **모니터링** | 가전별 조리 실패 VOC | 앱 내 '조리 결과 피드백' 기능의 부정적 의견 비율 |

## 3. 사용자 및 이해관계자
* **대상 사용자:** 빵지순례족, 1인 가구, 건강 관리자.
* **제외 대상:** 빵을 직접 베이킹하는 전문가(본 서비스는 소비 최적화에 집중).
* **이해관계자:** 제휴 베이커리(브랜드 경험 관리), 가전 제조사(기기 활용도 증대), 콘텐츠 크리에이터.

## 4. 정책 정의
* **기본 정책:** 모든 가이드는 빵의 수분 함량과 글루텐 구조를 기반으로 한 표준 레시피를 우선 제공함.
* **상태값 정의:**
    * `보관 상태`: 실온, 냉장(권장 안 함), 냉동, 해동 중.
    * `조리 상태`: 대기, 조리 중, 완료.
* **예외 정책:**
    * **API 타임아웃:** 외부 데이터(날씨, 레시피 DB) 호출 시 3초 이내 응답 없으면 로컬 캐시 데이터 우선 노출.
    * **결제 실패(Phase 3):** 결제 시도 시 PG사 응답 지연 발생 시 '재시도' 버튼 노출 및 5분 후 자동 재시도 로직 수행.

## 5. 서비스 구조 (유저 플로우)
1. **진입:** 앱 실행 → 빵 종류 선택(검색/카테고리)
2. **가이드 확인:** 보유 가전 선택 → 최적 온도/시간 확인
3. **실행:** '타이머 시작' 버튼 클릭 → 조리 시작
4. **완료:** 조리 완료 알림 → 결과 피드백(성공/실패) 입력

## 6. 상세 요구사항
| ID | 기능명 | 우선순위 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 빵 가이드 딕셔너리 | P0 | 20종 빵별 보관/조리법 DB 구축 및 조회 |
| REQ-02 | 스마트 타이머 | P0 | 선택한 가이드와 연동된 자동 타이머 기능 |
| REQ-03 | 가전별 설정값 필터 | P1 | 에어프라이어, 오븐, 토스터 등 기기별 설정값 제공 |
| REQ-04 | 결과 피드백 | P2 | 조리 결과 만족도 조사 및 가이드 보정 데이터 수집 |

* **운영 요구사항:** 관리자 페이지를 통해 가이드 DB 실시간 업데이트 및 신규 빵 종류 추가 기능.
* **데이터 요구사항:** 사용자별 조리 이력 로그 수집(가전 모델, 빵 종류, 만족도).

## 7. UX/UI 고려사항
* **로딩:** 가이드 로딩 시 빵이 구워지는 애니메이션(Lottie) 적용.
* **실패:** 네트워크 오류 시 '오프라인 모드(최근 조회 가이드)' 자동 전환.
* **빈 화면:** 검색 결과 없을 시 '가이드 요청하기' 버튼 노출.

## 8. 기술 고려사항
* **보안:** 사용자 입력 데이터(가전 모델 등)는 개인정보 비식별화 처리.
* **성능:** 이미지 최적화(WebP 포맷 사용)를 통해 가이드 로딩 속도 1초 이내 유지.

## 9. 실험 및 검증
* **A/B 테스트:** 타이머 UI 방식(슬라이더 vs 버튼)에 따른 조리 완료율 비교.
* **검증:** 초기 1,000명 대상 FGT(Focus Group Test)를 통해 가이드 정확도 검증.

## 10. 리스크 및 대응
* **리스크:** 가전 모델별 성능 편차로 인한 조리 실패.
* **대응:** 사용자 피드백 기반 '가이드 보정 알고리즘' 적용 및 모델별 커스텀 설정값 저장 기능 제공.
