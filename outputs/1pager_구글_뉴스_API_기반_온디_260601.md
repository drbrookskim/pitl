# [24.05.22] 구글 뉴스 API 기반 온디맨드 뉴스 큐레이션 플랫폼 (Product Team)

## 1. 프로젝트 한 줄 요약
정보 과잉 시대, 사용자의 관심사와 맥락에 맞춰 구글 뉴스 API를 실시간 필터링 및 요약하여 제공하는 '개인 맞춤형 온디맨드 뉴스 브리핑 서비스'

## 2. 대상
| 대상 | 상황/특징 |
| :--- | :--- |
| 2030 직장인 | 출퇴근 시간 내 핵심 이슈를 빠르게 파악하고 싶은 니즈 |
| 의사결정권자 | 특정 산업군/경쟁사 동향을 실시간으로 모니터링해야 하는 니즈 |
| 정보 피로군 | 넘쳐나는 뉴스 속에서 가짜 뉴스와 광고를 제외한 핵심 정보만 원하는 사용자 |

## 3. 문제 정의
*   **고객 문제:** 뉴스 앱의 광고 범람, 파편화된 정보, 관심 없는 분야의 기사 노출로 인한 정보 피로도 증가.
*   **비즈니스 문제:** 기존 뉴스 플랫폼의 낮은 체류 시간 및 광고 수익 모델의 한계.
*   **데이터 기반 현황:**
    | 지표 | 수치 | 비고 |
    | :--- | :--- | :--- |
    | 평균 뉴스 앱 이탈률 | 68% | 3분 내 앱 종료 |
    | 정보 탐색 소요 시간 | 25분/일 | 사용자 평균 뉴스 소비 시간 |
    | 광고 클릭률(CTR) | 0.8% | 뉴스 앱 평균 대비 저조 |
*   **Root Cause:** 사용자의 '관심사'와 '실시간 뉴스' 사이의 매칭 알고리즘 부재 및 정보 요약 기능의 결여.

## 4. 목표
| 항목 | 목표 수치 |
| :--- | :--- |
| 일간 활성 사용자(DAU) | 50,000명 달성 |
| 평균 체류 시간 | 12분/일 (현재 대비 4배 증가) |
| 뉴스 요약 만족도 | 4.5/5.0 (설문 기준) |
| 유료 구독 전환율 | 5% (프리미엄 요약 기능) |

## 5. 현황 및 분석
*   **VOC 인용:** "아침마다 뉴스 앱을 켜지만, 광고를 닫느라 시간을 다 씁니다. 내가 관심 있는 분야만 딱 요약해서 보여주는 서비스가 절실해요." (32세, IT 기획자)
*   **분석:** 현재 시장은 '공급자 중심'의 뉴스 나열 방식임. 사용자는 '큐레이션'과 '요약'이라는 가치에 기꺼이 비용을 지불할 의사가 있음.

## 6. 해결방안
*   **Phase 1 (MVP):** 구글 뉴스 API 연동 및 사용자 관심 키워드 기반 실시간 뉴스 피드 구성.
*   **Phase 2 (AI 고도화):** LLM(GPT-4o 등)을 활용한 뉴스 3줄 요약 및 핵심 인사이트 추출 기능 도입.
*   **Phase 3 (개인화):** 사용자 읽기 패턴 학습을 통한 '나만의 뉴스레터' 자동 생성 및 푸시 알림 최적화.

## 7. 리스크
| 영향도 | 리스크 | 대응 방안 |
| :--- | :--- | :--- |
| 상 | 구글 API 정책 변경 | 다중 뉴스 소스(RSS, Bing News 등) 확보 및 API 다변화 |
| 중 | 저작권 이슈 | 요약본 제공 시 원문 링크 필수 표기 및 트래픽 유도 |
| 하 | AI 요약 정확도 | 사용자 피드백 기반 요약 모델 파인튜닝 및 검수 로직 강화 |

## 8. 중요도 및 긴급도
*   **중요도: 상** (정보 과잉 시대의 필수적인 '필터링' 가치를 제공하므로 시장 경쟁력이 높음)
*   **긴급도: 상** (AI 뉴스 요약 서비스가 초기 시장을 형성하고 있어, 선점 효과를 위해 빠른 MVP 출시가 필요함)
