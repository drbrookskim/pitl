# [24.05.22] 구글 뉴스 API 기반 온디맨드 뉴스 큐레이션 플랫폼 PRD

## 0. 문서 개요
본 문서는 정보 과잉 시대에 사용자의 관심사에 맞춘 실시간 뉴스 필터링 및 AI 요약 서비스를 제공하는 '온디맨드 뉴스 큐레이션 플랫폼'의 상세 요구사항을 정의합니다.

## 1. 목표 (Goal)
*   **프로젝트 목표:** 사용자 맞춤형 뉴스 큐레이션 및 AI 요약을 통해 정보 탐색 효율을 극대화하고, 체류 시간 증대 및 유료 구독 모델을 안착시킴.
*   **성공 기준:**
    | 항목 | 목표 수치 |
    | :--- | :--- |
    | DAU | 50,000명 |
    | 평균 체류 시간 | 12분/일 |
    | 뉴스 요약 만족도 | 4.5/5.0 |
    | 유료 구독 전환율 | 5% |

## 2. KPI 및 측정지표
*   **핵심 KPI:** DAU, 평균 체류 시간, 유료 구독 전환율
*   **보조 지표:** 뉴스 클릭률(CTR), 요약본 공유 횟수, 푸시 알림 오픈율
*   **모니터링 지표:** API 호출 성공률, LLM 응답 지연 시간(Latency), 서버 에러율(5xx)

## 3. 사용자 및 이해관계자
*   **대상 사용자:** 2030 직장인, 산업군 모니터링이 필요한 의사결정권자, 정보 피로감을 느끼는 일반 사용자
*   **제외 대상:** 실시간성이 중요하지 않은 뉴스 소비자, 광고 기반의 무료 뉴스 앱 선호자
*   **이해관계자:** Product Team(기획/운영), Engineering(백엔드/AI), Design(UI/UX), Legal(저작권 검토)

## 4. 정책 정의
*   **기본 정책:** 모든 요약본은 원문 링크를 반드시 포함하며, 저작권 준수를 위해 요약은 핵심 내용 전달에 집중함.
*   **상태값 정의:**
    | 상태 | 설명 |
    | :--- | :--- |
    | PENDING | 뉴스 수집 및 요약 대기 중 |
    | COMPLETED | 요약 완료 및 피드 노출 |
    | FAILED | API 타임아웃 또는 요약 실패 |
*   **예외 정책:**
    | 상황 | 대응 정책 |
    | :--- | :--- |
    | API 타임아웃 | 재시도(Retry) 3회 후, 원문 제목만 노출 |
    | 결제 실패 | 결제 수단 변경 안내 팝업 노출 및 기능 제한 |
    | 요약 모델 오류 | 기본 뉴스 제목/본문 요약 없이 원문 링크만 제공 |

## 5. 서비스 구조 (User Flow)
1. 온보딩(관심 키워드 설정) → 2. 메인 피드(개인화 뉴스 리스트) → 3. 상세 보기(AI 요약본 확인) → 4. 원문 이동(선택) → 5. 구독(프리미엄 기능)

## 6. 상세 요구사항
| ID | 기능명 | 우선순위 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 뉴스 수집 | P0 | 구글 뉴스 API 연동 및 키워드 필터링 |
| REQ-02 | AI 요약 | P0 | LLM(GPT-4o) 기반 3줄 요약 생성 |
| REQ-03 | 개인화 피드 | P1 | 사용자 관심사 기반 뉴스 랭킹 알고리즘 |
| REQ-04 | 유료 구독 | P1 | 프리미엄 요약 기능 결제 및 구독 관리 |

*   **운영 요구사항:** 관리자 대시보드(API 호출량 모니터링, 사용자 피드백 관리)
*   **데이터 요구사항:** 사용자 클릭 로그, 체류 시간, 요약 만족도 평가 데이터 수집

## 7. UX/UI 고려사항
*   **로딩:** 뉴스 요약 생성 시 스켈레톤 UI 적용 (체감 대기 시간 단축)
*   **실패:** "요약을 불러올 수 없습니다. 원문을 확인하세요" 메시지 및 재시도 버튼 제공
*   **빈 화면:** "관심 키워드에 맞는 뉴스가 없습니다. 키워드를 추가해보세요" 안내 및 키워드 설정 유도

## 8. 기술 고려사항
*   **API 보안:** API Key 환경 변수 관리 및 호출 제한(Rate Limiting) 설정
*   **성능:** 뉴스 데이터 캐싱(Redis)을 통한 응답 속도 최적화
*   **보안:** 사용자 개인정보 암호화 및 결제 정보 토큰화 처리

## 9. 실험 및 검증
*   **A/B 테스트:** 요약 스타일(간결형 vs 상세형)에 따른 체류 시간 비교
*   **검증:** 베타 테스트 그룹 대상 요약 정확도 정성 평가(FGI)

## 10. 리스크 및 대응
*   **구글 API 정책 변경:** 다중 뉴스 소스(Bing News, RSS) 연동 모듈 설계
*   **저작권 이슈:** 법무 검토를 통한 '공정 이용' 가이드라인 준수 및 원문 트래픽 유도
*   **AI 요약 오류:** 할루시네이션 방지를 위한 프롬프트 엔지니어링 및 검수 로직 강화
