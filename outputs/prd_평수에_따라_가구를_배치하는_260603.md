# [24.05.22] AI 기반 평수 맞춤형 가구 배치 최적화 서비스 PRD

## 0. 문서 개요
*   **프로젝트 요약:** 사용자의 주거 평수 및 도면 데이터를 기반으로 가구 배치 시뮬레이션을 제공하여 구매 확신을 높이고 반품률을 낮추는 서비스.
*   **배경:** 가구 커머스 내 단순 상품 나열 방식의 한계로 인한 구매 결정 지연 및 사이즈 미스로 인한 높은 반품률(18%) 해결 필요.

## 1. 목표 (Goal)
*   **프로젝트 목표:** 공간 시뮬레이션 도구 도입을 통한 고객 경험 혁신 및 구매 전환율 증대.
*   **성공 기준:**
    | 항목 | 목표 수치 | 비고 |
    | :--- | :--- | :--- |
    | MAU | 5만 명 | 서비스 런칭 3개월 내 |
    | 구매 전환율(CVR) | 기존 대비 +15% | 시뮬레이션 이용자 대상 |
    | 반품률 | 18% → 8% 이하 | 사이즈/배치 불만족 사유 중심 |

## 2. KPI 및 측정지표
*   **핵심 KPI:** 구매 전환율(CVR), 반품률(Return Rate)
*   **보조 지표:** 시뮬레이션 완료율, 장바구니 담기 전환율, 평균 체류 시간
*   **모니터링 지표:** API 응답 속도(Latency), 서비스 에러율(5xx), 도면 데이터 입력 이탈률

## 3. 사용자 및 이해관계자
*   **대상 사용자:** 1인 가구, 신혼부부, 이사 예정자, 인테리어 초보자
*   **제외 대상:** 상업용 공간(오피스/카페) 인테리어 수요자 (Phase 1 제외)
*   **이해관계자:** 상품 기획팀(데이터 확보), 개발팀(엔진 구현), 디자인팀(UI/UX), CS팀(반품 대응)

## 4. 정책 정의
*   **기본 정책:** 모든 가구 데이터는 실제 규격(mm 단위)을 기준으로 하며, 배치 시 최소 동선(600mm) 확보를 권장함.
*   **상태값 정의:**
    | 상태값 | 설명 |
    | :--- | :--- |
    | DRAFT | 시뮬레이션 진행 중 |
    | COMPLETED | 배치 완료 및 저장 상태 |
    | VALIDATED | AI 검증 완료(동선/간섭 없음) |
*   **예외 정책:**
    | 상황 | 대응 정책 |
    | :--- | :--- |
    | 결제 실패 | 결제 재시도 안내 및 장바구니 유지 |
    | API 타임아웃 | 3초 이상 지연 시 '잠시 후 다시 시도' 안내 및 캐시 데이터 로드 |
    | 데이터 오차 | 사용자 직접 치수 보정 모드 강제 전환 |

## 5. 서비스 구조 (유저 플로우)
1. **진입:** 가구 상세 페이지 내 '내 방에 배치해보기' 버튼 클릭
2. **입력:** 평수 선택 및 도면(또는 직접 치수) 입력
3. **배치:** AI 추천 배치 또는 수동 배치
4. **검증:** 동선 간섭 및 사이즈 적합성 AI 피드백 확인
5. **구매:** 배치된 가구 리스트 장바구니 담기 및 결제

## 6. 상세 요구사항
| ID | 기능명 | 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 도면 입력 | 평수 선택 및 가로/세로 치수 입력 기능 | P0 |
| REQ-02 | 2D 시뮬레이터 | 가구 드래그 앤 드롭 및 회전 기능 | P0 |
| REQ-03 | AI 추천 배치 | 공간 효율 최적화 자동 배치 알고리즘 | P1 |
| REQ-04 | 동선 검증 | 가구 간 최소 거리(600mm) 미달 시 경고 | P1 |
| REQ-05 | 원스톱 결제 | 시뮬레이션 내 배치된 상품 일괄 장바구니 담기 | P0 |

*   **운영 요구사항:** 가구 DB 관리자 페이지(3D 모델링 업로드 및 규격 수정)
*   **데이터 요구사항:** 사용자별 배치 데이터 로그 수집(선호 가구, 배치 패턴)

## 7. UX/UI 고려사항
*   **로딩:** 시뮬레이터 로딩 시 '공간 최적화 중...' 프로그레스 바 노출
*   **실패:** API 호출 실패 시 '네트워크 상태를 확인해주세요' 토스트 메시지
*   **빈 화면:** 도면 데이터가 없을 경우 '기본 평면도 예시' 제공

## 8. 기술 고려사항
*   **렌더링:** 웹 기반 WebGL 라이브러리(Three.js 등) 활용하여 경량화
*   **보안:** 사용자 도면 데이터 암호화 저장 및 개인정보 비식별화 처리
*   **성능:** 가구 3D 모델링 파일(GLB/GLTF) 최적화(폴리곤 수 제한)

## 9. 실험 및 검증
*   **A/B 테스트:** 'AI 자동 배치 추천' 기능 노출 여부에 따른 구매 전환율 비교
*   **사용성 테스트(UT):** 5인 대상 도면 입력 과정의 이탈 지점 분석

## 10. 리스크 및 대응
*   **도면 오차:** 사용자 직접 보정 가이드 및 튜토리얼 영상 제공
*   **DB 확보 지연:** 주요 브랜드 5개사 우선 제휴 및 범용 가구 모델링 데이터 활용
*   **기술 부하:** 클라이언트 사이드 렌더링을 통해 서버 부하 최소화
