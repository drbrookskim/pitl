# [24.10.24] 가치 투자자를 위한 스마트 저평가 우량주 발굴 플랫폼, 'ValueFinder' PRD

## 0. 문서 개요
본 문서는 복잡한 재무제표 분석 없이 개인 투자자가 3초 만에 저평가 우량주를 발굴하고 포트폴리오를 진단할 수 있는 'ValueFinder'의 MVP 개발을 위한 상세 요구사항 명세서입니다.

## 1. 목표 (Goal)
* **프로젝트 목표:** 데이터 기반의 직관적인 필터링 엔진과 시각화 도구를 통해 개인 투자자의 종목 분석 진입장벽을 제거하고, 가치 투자 문화를 정착시킴.
* **성공 기준:**
| 항목 | 목표 수치 |
| :--- | :--- |
| 종목 탐색 시간 | 4.2시간 → 3분 이내 단축 |
| D+7 잔존율 | 45.0% 이상 |
| 유료 구독 전환율 | 3.5% 달성 |

## 2. KPI 및 측정지표
* **핵심 KPI:** D+7 잔존율, 유료 구독 전환율, 종목 탐색 완료율(필터링 시작 후 결과 확인까지의 비율)
* **보조 지표:** 일간 활성 사용자 수(DAU), 평균 세션 시간, 필터링 엔진 사용 횟수
* **모니터링 지표:** API 호출 성공률, 서버 응답 시간(Latency), 결제 실패율

## 3. 사용자 및 이해관계자
* **대상 사용자:** 재무제표 분석에 어려움을 겪는 직장인, 가치투자 입문자, 감정적 매매를 지양하는 투자자.
* **제외 대상:** 단기 급등주/테마주 위주의 초단타 매매자(스캘퍼).
* **이해관계자:** Product Team(기획/디자인), 개발팀(FE/BE/Data), 법무팀(면책 조항 검토), 마케팅팀.

## 4. 정책 정의
* **기본 정책:** 모든 투자 정보는 참고용이며, 최종 결정은 사용자 본인에게 있음(면책 고지 필수).
* **상태값 정의:**
| 상태값 | 설명 |
| :--- | :--- |
| 신호등(Green) | 우량/저평가 구간 (매수 고려) |
| 신호등(Yellow) | 중립/관망 구간 |
| 신호등(Red) | 고평가/위험 구간 (주의 필요) |
* **예외 정책:**
| 상황 | 처리 정책 |
| :--- | :--- |
| API 타임아웃 | 3초 이상 지연 시 '데이터 로딩 실패' 메시지 노출 및 재시도 버튼 제공 |
| 결제 실패 | 결제 수단 오류 시 즉시 알림 및 결제 수단 변경 유도 |
| 데이터 미비 | 상장 폐지/거래 정지 종목은 필터링 결과에서 자동 제외 |

## 5. 서비스 구조 (User Flow)
1. **온보딩:** 투자 성향 선택 (배당/성장/저평가)
2. **메인:** 3초 퀵-필터 실행
3. **결과:** 종목 리스트 및 신호등 점수 확인
4. **상세:** 개별 종목 재무 지표 시각화 확인
5. **전환:** 프리미엄 기능(알림/포트폴리오 진단) 구독 유도

## 6. 상세 요구사항
| ID | 기능명 | 우선순위 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| REQ-01 | 퀵-필터 엔진 | P0 | 성향별 상위 10개 종목 추출 알고리즘 구현 |
| REQ-02 | 신호등 스코어링 | P0 | 4개 영역(수익/안전/성장/저평가) 점수화 및 시각화 |
| REQ-03 | 구독 결제 모듈 | P1 | 인앱 결제 및 정기 구독 관리 연동 |
| REQ-04 | 면책 고지 팝업 | P0 | 앱 진입 시 필수 동의 절차 구현 |

* **운영 요구사항:** 관리자 페이지를 통한 종목별 신호등 가중치 조정 기능.
* **데이터 요구사항:** 일일 재무 데이터 업데이트 배치 작업(Batch) 및 캐싱 전략.

## 7. UX/UI 고려사항
* **로딩:** 3초 이내 완료를 위해 스켈레톤 UI 적용.
* **실패:** "데이터를 불러오는 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요." 문구와 함께 재시도 버튼 배치.
* **빈 화면:** 검색 결과가 없을 경우 "현재 조건에 맞는 종목이 없습니다. 필터를 완화해보세요." 가이드 제공.

## 8. 기술 고려사항
* **보안:** 금융 데이터 API 호출 시 API Key 암호화 및 환경 변수 관리.
* **성능:** Redis를 활용한 종목별 스코어링 데이터 캐싱으로 API 호출 비용 및 응답 속도 최적화.

## 9. 실험 및 검증
* **A/B 테스트:** 필터링 결과 화면의 UI 구성(리스트형 vs 카드형)에 따른 클릭률 비교.
* **검증:** 백테스팅 데이터를 활용하여 신호등 점수와 실제 주가 흐름의 상관관계 주기적 검증.

## 10. 리스크 및 대응
* **리스크:** 추천 종목의 수익률 저조로 인한 신뢰도 하락.
* **대응:** 과거 데이터 기반 백테스팅 결과 투명 공개 및 '투자 책임은 본인에게 있음'을 UI 곳곳에 명시.
